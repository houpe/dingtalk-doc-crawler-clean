# 使用说明

钉钉文档一条龙 Pipeline 的快速上手指南。完整介绍见 [README.md](README.md)，工作流详解见 [docs/workflow.md](docs/workflow.md)。

## 目录结构

```
dingtalk-doc-crawler-clean/
├── src/                        # Python 脚本
│   ├── pipeline.py             # 一条龙主入口（抓取→过滤→优化→VitePress 构建）
│   ├── dws_crawler.py          # Stage 1: dws CLI 抓取钉钉文档 + 下载图片
│   ├── reformat_md.py          # Stage 3: MD 规则优化
│   ├── optimize_md_deepseek.py # Stage 3 可选: DeepSeek AI 语义优化
│   ├── post_process.py         # Stage 4: 构建前兜底（清 HTML / 修图链 / 删死链）
│   ├── build_vitepress.py      # 从 output_optimized 直接构建
│   ├── preview_vitepress.py    # 从 output_optimized 直接预览
│   └── download_images_only.py # 只补下载图片，不重抓文档
├── assets/                     # 静态资源
│   └── favicon.png             # 网站图标
├── requirements.txt            # Python 依赖
├── output/                     # 原始抓取（gitignore）
├── output_optimized/           # 规则优化后（gitignore）
├── output_reformatted/         # AI 模式中间产物（gitignore）
├── site/                       # VitePress 站点（源 + 构建产物）
│   ├── docs/                   # 文档源 + .vitepress/ 配置
│   ├── server.js               # Express 后端 + 在线编辑（pipeline 生成）
│   └── package.json
├── scripts/
│   └── copy_images.py          # 辅助：复制图片到 site/docs
├── docs/                       # 设计 / 计划 / 工作流文档
└── tests/
```

## 前置准备

1. **Python 依赖**

   ```bash
   pip install -r requirements.txt
   ```

2. **dws CLI**（仅 Stage 1 抓取需要）—— 需自行安装并登录，确保 `dws doc list` 可用。已有 `output/` 时可跳过。

3. **Node / npm**（Stage 4 VitePress 构建需要）—— 首次构建时 pipeline 会自动 `npm install`。

## 快速使用

### 全量一条龙（抓取 → 过滤 → 优化 → 构建预览）

```bash
python3 src/pipeline.py "钉钉文档分享URL"
```

### 最常用：从本地 output 重建

```bash
python3 src/pipeline.py --source ./output
```

### 构建后直接部署

```bash
python3 src/pipeline.py --source ./output --deploy fast   # 只传非图片（快）
python3 src/pipeline.py --source ./output --deploy full   # 全量重传
```

### 启用 AI 语义优化（默认关闭）

```bash
python3 src/pipeline.py --source ./output --use-ai
```

### 单独补下载图片（不重抓文档）

```bash
python3 src/download_images_only.py
```

### 单独预览 VitePress 站点

```bash
cd site && npx vitepress dev docs --port 4001
```

## 更新文档的标准流程

```bash
# 1. 需要重新拉取时（首次或钉钉侧有更新）
python3 src/dws_crawler.py

# 2. 全流程重建
python3 src/pipeline.py --source ./output

# 3. 浏览器预览（pipeline 默认会启动 http://localhost:4000）
#    或单独预览：
cd site && npx vitepress dev docs --port 4001

# 4. 确认无误后部署
python3 src/pipeline.py --source ./output --deploy fast
```

## 在线编辑

`site/server.js`（pipeline 生成）提供带登录的在线编辑：

```bash
node site/server.js
# 文档站点:   http://localhost:4000
# 编辑入口:   http://localhost:4000/edit.html
# 默认账号:   admin / admin123（环境变量 ADMIN_USER / ADMIN_PASS 可覆盖）
```

登录后可在 Monaco 编辑器里直接改 Markdown、保存并触发 VitePress 重建。已登录用户在文档页右下角会出现「✏️ 编辑」悬浮按钮。
