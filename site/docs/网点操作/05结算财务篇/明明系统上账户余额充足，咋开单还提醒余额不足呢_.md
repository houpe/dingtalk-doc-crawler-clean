# 明明系统上账户余额充足，咋开单还提醒余额不足呢

## 一、适用场景

本文适用于网点用户开单时系统提示“余额不足”，但账户总余额显示充足的情况。帮助您理解 **账户余额** 与 **可用余额** 的区别，并排查差异原因。

## 二、前置条件

- 已开通对应菜单权限。
- 已登录系统并进入账户余额查询页面。

## 三、操作入口

请以系统实际菜单路径为准（通常可在“资金管理”或“账户中心”下找到余额查询入口）。

## 四、操作步骤

1. 在系统中查看 **账户余额**（总余额）与 **可用余额** 的差异。
   - **账户余额**：网点在系统中的总余额。
   - **可用余额**：网点当前可实际使用（录单、物料申购、缴纳包仓费、提现等）的余额。
2. 若两者不一致，差异金额会展示在 **冻结金额** 中。点击冻结金额数值，可查看详情。

   ![查看冻结金额详情](./images/包仓仓位用的好快，哪里能看到仓位的使用明细呢？_1_4b707d03.png)

3. 了解产生冻结金额的常见业务场景：
   - **发起提现申请后**：流程未审批完结前，提现金额会冻结。
   - **作为付款方收到的费用申请**：该申请处理完结之前，对应金额会冻结。

## 五、操作结果

操作完成后，可以理解余额不足的真正原因（为冻结金额占用），并根据冻结明细联系相关人员处理提现审批或费用申请，释放可用余额。

## 六、注意事项

::: danger 重点提醒
开单时系统校验的是 **可用余额**（即总余额减去冻结金额），而非账户总余额。请以可用余额作为操作判断依据。
:::
