# 差错如何申报申诉？

## 一、适用场景

本文适用于网点、中心、云仓或网络货运人员，当发现运单差错需要向责任方申报，或收到差错申报后需要发起申诉时，按此文档操作。

## 二、前置条件

- 账号已开通**品控工作台**对应权限（流程节点人员可操作，非流程节点人员仅可查询）。
- 申报方在接收到差错后**72小时内**可发起申诉，超时则自动流转至总部。

## 三、操作入口

通过菜单路径：**服务质量 → 品控工作台**（具体路径以系统实际菜单为准）。

- **流程节点人员**：在【操作】中处理或查看数据。
- **非流程节点人员**：在【查询】中查看数据。

## 四、操作步骤

### 4.1 查询差错数据

进入品控工作台页面，可查看所有差错记录。列表支持按条件筛选、查看详情。

![image.png](./images/仲裁如何申报、申诉？_10_2674e296.png)

### 4.2 发布差错申报

在页面点击**新增（发布差错）**按钮，填写以下信息：

| 字段 | 说明 |
|------|------|
| **差错编号** | 自动生成，规则：年月日时分秒各2位 + c + 随机数5位 |
| **运单号/调度单号** | 非总部填写时必填 |
| **异常数** | 非必填；若填写运单号，异常数不可大于运单总件数 |
| **差错类型** | 必选 |
| **差错类别** | 必选 |
| **问题描述** | 非必填 |
| **附件** | 非必填 |

::: warning 注意事项
- 已签收运单**不可申报**。
- 同一运单若存在总部**不受理**的申报数据，则可再次申报；否则不可再次申报。
:::

![image.png](./images/仲裁如何申报、申诉？_11_14378a46.png)

### 4.3 发起差错申诉

只有当差错记录状态为**待处理**时，才能点击**申诉**按钮进行申诉。

- 系统会校验当前申报单状态，若状态不是**已申报**，则提示：“申诉失败！当前差错已在，不可申诉，有疑问请联系总部运营-曲丽娜”。

![image.png](./images/仲裁如何申报、申诉？_1_575eadf9.png)

## 五、操作结果

- 申报提交后，记录进入**已申报**状态，等待被申报方在72小时内申诉。
- 被申报方申诉后，系统自动流转至总部处理。
- 总部处理结果：**不受理**——不产生罚款，申报方可在同一运单再次申报；**受理**——产生罚款，当前运单不可再发起申诉。

## 六、注意事项

- 申报方申报后，被申报方未在**72小时内**申诉的，自动流转至总部。
- 总部选择**不受理**时，申报方可针对该运单再次申报；总部受理后，该运单禁止再次申诉。
- **已签收运单**不可发起申报。

## 七、常见问题

**Q：为什么申诉按钮是灰色不可点击？**
A：只有差错记录状态为“待处理”时才能申诉。若显示其他状态（如已处理、已仲裁等），请联系总部运营确认。
