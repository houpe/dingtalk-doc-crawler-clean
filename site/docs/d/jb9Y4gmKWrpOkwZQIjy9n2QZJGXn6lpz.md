# PC端怎么查看网点相关数据

## 一、适用场景

本文适用于网点管理人员在 **鲸天系统 PC端** 查看网点首页数据，进行日常运营管理、异常督办、提派资源规划、资金盘点和核心指标监控。

网点首页是日常运营管理的核心“驾驶舱”。页面通过汇总业务量、运营风险与财务状况等数据，帮助网点管理者一站式完成异常事项的“日清”督办，并实时关注资金池与核心指标健康度，保障整体运营通畅。

## 二、前置条件

::: danger 重点提醒
需具备 **网点权限** 才能查看网点首页相关数据，**业务员无权限查看**。
:::

操作前请确认：

1. 已具备 **鲸天系统 PC端** 登录账号。
2. 账号已开通对应 **网点权限**。
3. 登录后当前组织为对应的 **xxx网点**。
4. 已准备好需要查看或核对的业务数据。

## 三、操作入口

登录 **鲸天系统 PC端**，查看当前组织是否为 **xxx网点**；默认展示页面即为 **【网点首页】**。

如页面入口与本文不一致，请以系统实际菜单路径为准。

## 四、操作步骤

### 4.1 了解首页核心数据

查看网点首页前，建议先理解以下核心模块含义：

- **待办**：系统汇集的亟需干预处理的异常事项，核心目标为“日清”（每天下班前需清零）。
- **延误红绿灯**：直观展示进出港运单流转时效的预警看板，用于快速锁定并拦截即将延误或已延误的运单。
- **预估进港待提货量**：系统根据上游路由节点测算的即将抵达本网点提货分拨的货量规模，用于提前安排并制定提货计划。

### 4.2 场景一：早班资源调度与资金盘点

1. 每日晨会前，优先查看 **【预估进港待提货量】**。
2. 根据预估提货量，提前联络车队安排运力，并合理排布派件路线。
3. 检查财务资金状况区，重点核对 **【账户余额】** 与 **【电子面单情况】**。
4. 如面单库存不足，或余额逼近警戒线，及时发起充值或采购申请，避免因欠费、缺单导致前端无法正常承接派件。

### 4.3 场景二：日间时效干预与风险监控

1. 高频巡查 **【延误红绿灯】** 区域。
2. 针对日常进出港运单，如出现 **黄灯** 或 **红灯** 告警，立即点击穿透至详情页。
3. 根据详情页信息，联动上下游网点进行催办。
4. 监控资金回流，查看 **【进港到付款情况(元)】**。
5. 当累计金额较大时，重点跟进并督促派件员及时回款。

## 五、操作结果

操作完成后，可在 **【网点首页】** 查看以下结果：

1. 可看到网点相关的运营、时效、资金等数据。
2. 可通过 **【预估进港待提货量】** 辅助安排提货与派件资源。
3. 可通过 **【延误红绿灯】** 识别并跟进黄灯、红灯风险运单。
4. 可通过 **【账户余额】**、**【电子面单情况】** 和 **【进港到付款情况(元)】** 关注资金与面单情况。
5. 对应模块支持点击跳转时，可进入明细页面进一步查看数据。

::: warning 注意事项
如页面展示、权限范围或业务规则与本文不一致，请以当前系统配置和最新业务规则为准。
:::

::: danger 重点提醒
- **待办** 的目标为“日清”，即 **每天下班前需清零**。
- 如打开首页看不到数据，优先确认账号是否具备对应 **网点权限**。

### 6.1 核心模块跳转及逻辑

- **订单与运单管理**
  - **待接单**：跳转至 **经营管理中心-订单管理-订单管理**。默认过滤 **近30日**、**待接单状态**、**当前查询网点** 的数据。
  - **待登记回单**：跳转至 **经营管理中心-运单管理-回单管理**。定位至 **派方回单** 页签，默认过滤 **未登记**、**当前查询网点**、**近30天签收** 的数据。
  - **待确认改单**：跳转至 **经营管理中心-运单管理-运单调整确认**。默认过滤 **未审核**、**近30天修改** 的数据。

- **费用管理（待处理申请）**
  - 跳转至 **财务管理-账单管理-费用申请**。

- **进港到付情况（优化项）**
  - **在途未签收** 与 **今日已签收** 均跳转至 **经营管理中心-运单管理-派件运单管理**。
  - 根据时间区间 **00:00:00-23:59:59** 和网点角色进行精细化过滤。

- **预估进港待提货量**
  - 细分为 **今天已到**、**今天未到**、**明天**、**后天** 四个维度。
  - 统一落地到 **进出港跟踪-进港** 页面。
  - 通过不同的 **跟踪状态** 及精确到具体日期的时间段参数（**0点-24点**）进行联动。

## 七、常见问题

### 7.1 Q1：首页看到的数据与点击跳转后页面的数据不一致，怎么办？

A：提交 **小吉工单**，由技术确认具体原因；同时可以以跳转后页面的数据作为明细基础重新统计。

### 7.2 Q2：打开首页看不到数据，怎么办？

A：通常属于权限问题。请先验证自己是否有该网点权限，可通过 **小吉工单** 申请权限；如果仍然无数据，可通过 **小吉工单** 进行异常上报，由技术排查原因。
