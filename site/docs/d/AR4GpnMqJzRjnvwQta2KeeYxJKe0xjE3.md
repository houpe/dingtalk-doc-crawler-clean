# 运输任务

## 适用场景

本文适用于干线/支线冷链运输司机使用 **中通冷链司机版微信小程序-车线任务** 模块，完成调度下发运输任务的线上操作，包括：

- **任务接单**
- **始发签到**
- **发车上传凭证**
- **在途节点到达**
- **分拨靠台拍照**
- **任务完结**
- **自主创建加油/维修等日常事务任务**
- **在途异常上报**
- **车辆冷机、油感硬件状态查看**
- **历史运输任务台账查询**

系统用于全程冷链数据留痕、节点闭环管控，保障货物温控合规、运输流程可追溯。

## 前置条件

1. **账号与权限**
   - 使用分配的司机手机号，通过微信登录 **中通冷链司机版** 小程序。
   - 车辆完成绑定后，自动开通 **车线任务** 相关功能权限。
   - 如看不到任务入口，请联系网点调度核查车辆绑定状态。

2. **设备与权限**
   - 准备可正常使用的智能手机。
   - 小程序需开启 **定位、相机、相册** 权限。
   - 定位异常可能导致任务不展示，或无法完成签到、靠台扫码等操作。

3. **配套工具**
   - 微信搜索进入 **中通冷链司机版** 小程序。

## 操作入口

- 小程序入口：微信搜索进入 **中通冷链司机版**
- 功能入口：底部导航栏点击 **车线任务**
- 主要标签页：
  - **待接受**
  - **进行中**
  - **已完成**

## 核心名词说明

- **车线任务**：小程序核心运输模块，分为 **待接受、进行中、已完成** 三大标签页，承载调度下发运输班线任务与司机自主创建事务任务。
- **待接受任务**：调度已下发、司机未启动的运输任务，分为固定班线、日常事务（加油）两类，支持滑动滑块开始任务或取消任务。
- **进行中任务**：已启动运输的在途任务，细分为 **已接受、已签到、已发车、已到车、已靠台** 状态，完整展示运输节点流程：签到 → 发车 → 到车 → 靠台 → 完成。
- **签到**：车辆抵达始发分拨中心后，线上确认到店的操作，是发车前置条件。
- **发车操作**：装车完毕后，上传车厢货物、车尾封签、冷机面板照片，并设置冷机运行状态与温层；提交后完成发车，要求发车后 **15分钟内** 驶离园区。
- **到达节点**：运输途经中转分拨中心时，滑动滑块完成节点到达确认，页面展示路线地图、运单票件、重量、体积明细。
- **靠台拍照**：车辆抵达终点分拨后，扫描或选择月台编号，拍摄合规现场照片并上传提交。照片需包含三角木、月台号、车牌等关键信息。
- **自主创建任务**：司机手动发起加油、洗车、维修、其他类日常事务任务，可选择随车协作司机。
- **异常上报**：运输途中发生故障、堵车、货物失温等突发情况时，上传现场照片并填写文字备注提交报备，最多上传 **3张** 图片。
- **硬件状态**：实时查看车辆 GPS 轨迹、双冷机运行温度、油感油量数据，用于识别冷机、油感设备异常预警。
- **已完成任务台账**：归档历史运输任务，展示计划/实际时长里程、晚点时长、异常次数、系统罚款、申诉入口、任务报告等信息。
- **冷链温控标准**：冷冻货物冷机设定 **≤-15℃**，冷藏货物 **0~10℃**；发车前冷机需预冷达标，系统无温度数据或温度不达标时禁止发车。

## 操作步骤

### 接收并启动待接受运输任务

**系统路径：** 小程序底部 **车线任务** → 顶部切换 **待接受**

1. 进入 **车线任务** 页面，切换至 **待接受** 标签。
2. 查看调度下发任务，任务类型包括：
   - **固定班线（干线运输）**
   - **日常事务-加油**
3. 核对任务信息，包括起终点、里程、计划时间、双驾司机、车牌信息等。
4. 确认无误后，向右滑动底部绿色滑块 **开始**。
5. 任务流转至 **进行中-已接受** 状态。
6. 如需取消任务，点击左侧 **取消** 放弃任务。

### 自主创建事务任务

**系统路径：** 待接受任务卡片 → **自主创建**

1. 点击任务卡片内 **自主创建** 按钮，进入创建任务页面。
2. 选择任务类型：
   - **加油**
   - **洗车**
   - **维修**
   - **其他**
3. 根据需要填写文字备注（选填）。
4. 选择随车司机：
   - **仅自己**
   - **同车协作司机**
5. 核对信息后，点击底部 **提交**。
6. 系统生成日常事务待接受任务。

### 始发分拨签到

**系统路径：** 车线任务 → **进行中** → **已接受** 任务卡片

1. 车辆抵达始发分拨园区后，在 **进行中** 列表找到目标任务。
2. 确认任务当前流程停留在 **签到** 节点。
3. 车辆定位匹配分拨园区后，系统自动完成签到。
4. 签到节点标记为对勾后，任务流转至下一步 **发车**。

::: danger 重点提醒
如页面出现红色提示栏 **温度设备未达5℃以下/无温度数据**，禁止发车。请先进入 **硬件状态** 页面排查冷机设备，预冷达标后再执行发车。
:::

### 发车提报

**系统路径：** 进行中 → **已签到** 任务 → 滑动 **发车** 滑块 → 弹出上传发车照片弹窗

1. 装车、封签全部完成后，向右滑动页面 **发车** 滑块。
2. 系统弹出 **上传发车照片** 弹窗。
3. 点击弹窗 **确认**，进入发车资料填写页面。
4. 按要求上传以下 **4类照片**：
   - **车厢货物照片**：拍摄车厢内完整堆货全貌。
   - **车厢尾部照片**：拍摄关闭后的车厢门和完整封签特写。
   - **冷机1面板照片**：清晰拍摄冷机屏幕实时温度数值。
   - **冷机2面板照片**：清晰拍摄冷机屏幕实时温度数值。
5. 配置冷机信息：
   - 开启或关闭对应冷机开关。
   - 勾选对应温层：**冷冻** 或 **冷藏**。
   - 双温车辆按货物流向分别设置。
6. 核对页面红色提示 **发车后请15分钟内离开园区**。
7. 确认所有照片上传完毕后，点击底部 **提交**。
8. 发车完成后，任务节点更新为 **已发车**。

::: danger 重点提醒
发车照片为发车提报的重要凭证。车厢货物、车尾封签、冷机面板照片需按页面要求上传。
:::

### 在途中转节点到达确认

**系统路径：** 进行中 → **已发车** 任务卡片

1. 行驶抵达中途中转分拨中心后，打开任务详情。
2. 查看页面展示的路线地图、起终点、中转站点、票件、件数、重量、体积明细。
3. 向右滑动底部 **到** 滑块。
4. 系统记录到达时间，完成中转节点到达登记。
5. 完成后继续下一程运输。

### 终点分拨靠台拍照提交

**系统路径：** 进行中 → **已到车** 任务 → 滑动 **靠台** 滑块 → 跳转靠台拍照页

1. 车辆驶入终点分拨指定月台。
2. 滑动页面 **靠台** 滑块，进入靠台拍照上传页面。
3. 在 **停靠月台** 栏，点击右侧扫码图标扫描现场月台二维码，或手动选择月台编号。

::: danger 重点提醒
带 **\*** 的月台编号为必填项。
:::

4. 上传靠台现场照片，照片需满足以下规范：
   - 三角木垫在主驾驶侧轮胎。
   - 月台编号完整清晰。
   - 车辆车牌清晰可见。
5. 照片上传完成后，核对月台信息。
6. 确认无误后，点击底部 **提交**。
7. 靠台操作完成后，流程进入最后 **完成** 节点。

### 任务完结

**系统路径：** 进行中 → **已靠台** 任务卡片

1. 整车卸车、交接全部完成后，进入对应 **已靠台** 任务卡片。
2. 向右滑动底部 **完成** 滑块。
3. 整单运输任务结束，并自动归档至 **已完成** 台账标签页。

### 在途异常上报

**系统路径：** 任意运输任务详情页 → 右侧 **异常上报** 悬浮按钮

1. 点击页面右侧红色三角 **异常上报**。
2. 进入现场照片上传页面。
3. 点击加号框上传现场佐证图片，最多上传 **3张**。
   - 车辆故障
   - 货物破损
   - 冷机异常实拍
4. 在备注输入框填写异常详情，备注上限 **50字**。
   - 示例：高速堵车预计晚点2小时
   - 示例：冷机故障温度超标
   - 示例：货物外包装破损
5. 信息填写完整后，点击 **提交**。
6. 调度后台实时接收异常报备记录。

### 查询硬件冷机、油感状态

**系统路径：** 车线任务任意页面，下拉或菜单入口进入 **硬件状态**

1. 进入 **硬件状态** 页面。
2. 查看页面三大模块：
   - **轨迹与温度**
   - **冷机状态**
   - **油感管理**
3. 查看顶部是否有红色标红提示；红色标红通常代表设备异常。
4. 在 **轨迹与温度** 中，查看实时 GPS 位置、双温传感器实时温度、行驶速度、总导航里程。
5. 在 **冷机状态** 中，查看单/双冷机启停模式、引擎运行时长、设定温度、出风/回风实时温度。
6. 如冷机状态显示 **异常**，可能代表冷机通讯故障、停机、温度不达标。
7. 在 **油感管理** 中，查看双油箱油量、累计行驶油量。
8. 如设备状态标红，可能代表油量传感器故障。
9. 点击页面右上角 **刷新**，同步最新车辆硬件实时数据。

### 查询已完成任务台账与申诉

**系统路径：** 车线任务 → 顶部切换 **已完成**

1. 切换至 **已完成** 标签。
2. 按任务年月筛选，查看所有归档历史运输任务。
3. 查看单条任务核心数据：
   - 计划时长
   - 实际时长
   - 在途晚点
   - 异常次数
   - 规划/绕道里程
   - 系统罚款金额
4. 如存在不合理罚款，点击罚款右侧 **申诉** 入口。
5. 上传凭证并提交申诉。
6. 点击右下角 **查看报告**，查看该趟运输的完整流程节点、温控、照片、异常记录报表。

## 操作结果

完成以上操作后，系统应显示对应任务状态或结果：

- 接收任务后，任务进入 **进行中-已接受**。
- 始发签到完成后，流程进入 **发车** 节点。
- 发车提交成功后，任务状态更新为 **已发车**。
- 中转到达登记后，系统记录中转节点到达时间。
- 靠台提交成功后，流程进入 **完成** 节点。
- 完结后，任务自动归档至 **已完成** 台账。
- 异常上报提交后，调度后台可接收异常报备记录。
- 已完成任务可查看任务报告，并在符合页面条件时发起 **申诉**。

## 注意事项

::: danger 重点提醒
- 发车前如系统提示 **无温度数据** 或 **温度设备未达5℃以下**，禁止强行发车。
- 发车后请在 **15分钟内** 离开园区。
- 异常上报图片最多上传 **3张**，备注上限 **50字**。
- 靠台月台编号带 **\*** 时为必填项。
- 冷冻货物冷机设定 **≤-15℃**，冷藏货物 **0~10℃**；发车前冷机需预冷达标。
:::

::: warning 注意事项
- 小程序需开启 **定位、相机、相册** 权限，否则可能影响任务展示、签到、扫码、拍照上传等操作。
- 靠台照片需确保三角木、月台编号、车牌清晰可见。
- 硬件状态页面如冷机、油感显示异常，请及时刷新并按实际情况上报。
- 如任务入口或任务数据异常，请联系网点调度或管理员核查。
:::

## 常见问题

### 常见异常与处理方案

| 序号 | 异常现象 / 报错提示 | 常见原因 | 解决方案 |
|------|---------------------------|------------|------------|
| 1 | 车线任务页面空白，无待接受/进行中任务 | 小程序定位权限关闭、车辆未绑定、调度未下发任务 | 1. 小程序右上角···-设置，开启位置权限；2. 联系网点管理员确认车辆已绑定司机账号；3. 对接调度确认任务已下发，下拉页面刷新。 |
| 2 | 发车页面提示「无温度数据，请检查温度设备」 | 车载冷机硬件离线、传感器故障、冷机未开机 | 1. 进入【硬件状态】刷新查看冷机是否在线；2. 重启车辆冷机设备，等待5分钟同步温度；3. 冷机持续异常走【异常上报】提交故障照片报备调度。 |
| 3 | 发车提示「温度设备未达5℃以下」 | 车厢预冷不足，冷藏/冷冻温度未达标 | 延长车辆预冷时间，待硬件页面温度达标后再执行发车操作，禁止强行发车。 |
| 4 | 靠台拍照无法扫描月台码 | 小程序无相机权限、二维码模糊、不在分拨月台定位范围 | 1. 开启小程序相机权限；2. 手动选择月台编号替代扫码；3. 车辆停靠至月台指定区域再操作。 |
| 5 | 发车/靠台照片上传失败 | 网络信号弱、图片体积过大 | 1. 切换园区稳定WiFi再上传；2. 重新拍摄压缩图片，保证画面清晰、大小适中。 |
| 6 | 滑动滑块（开始/发车/到达/靠台/完成）无响应 | 小程序缓存卡顿、定位未匹配站点、任务状态异常 | 1. 彻底关闭小程序后台重新进入；2. 确认车辆已抵达对应分拨园区，等待定位刷新；3. 联系调度后台重置任务状态。 |
| 7 | 硬件状态页面冷机/油感显示异常 | 车载终端断电、传感器线路故障、信号丢失 | 1. 点击页面右上角刷新按钮多次同步；2. 停车检查冷机、油感设备供电线路；3. 同步异常上报告知调度安排维修。 |
| 8 | 自主创建任务提交后不显示在待接受列表 | 任务类型填写缺失、网络提交未成功 | 1. 切换网络重新提交创建任务；2. 下拉待接受页面刷新列表。 |
| 9 | 已完成任务存在罚款，申诉入口无法点击 | 任务未归档完成、申诉时效过期 | 1. 确认该趟任务所有节点全部提交完结；2. 超过申诉时效联系网点线下提交纸质申诉材料。 |
| 10 | 异常上报无法提交备注/图片 | 输入框字符超限、图片数量超出3张上限 | 1. 精简备注文字控制在50字以内；2. 删除多余图片，最多保留3张现场实拍图再提交。 |

### 发车上传照片有硬性要求吗？不传可以提交吗？

车厢货物、车尾封签、双冷机面板照片全部为 **必填项**。缺少任意一张，无法点击提交发车。

发车后必须 **15分钟内** 驶离分拨园区，超时会产生晚点记录与系统罚款。

### 冷机一、冷机二分别怎么设置温层？

- 单温冷藏车：仅开启 **冷机1**，勾选 **冷藏**。
- 单温冷冻车：勾选 **冷冻**。
- 双温车厢车辆：冷冻区域对应冷机设置冷冻，冷藏区域对应冷机设置冷藏，按实际装载货物温区选择。

### 靠台拍照照片拍摄标准是什么？

靠台照片需满足以下要求：

1. 主驾轮胎两侧垫放三角木，并完整入镜。
2. 月台墙面编号清晰可见。
3. 车辆完整车牌拍摄清楚。

如照片不满足规范，可能会被后台驳回，需要重新上传。

### 运输途中遇到堵车、车辆故障、货物失温必须上报吗？

必须通过页面右侧 **异常上报** 提交现场照片和文字说明。

未报备产生晚点、货损罚款由司机自行承担；提前上报调度可协调调整时效、安排维修，并按实际情况处理对应处罚。

### 硬件状态页面温度数据多久刷新一次？

硬件状态数据实时刷新。点击右上角 **刷新** 按钮，可手动同步最新车载冷机、GPS、油量数据。

如长时间无更新，可能代表车载设备离线，需立即报备调度检修。

### 自主创建的加油任务和调度下发班线任务操作流程一样吗？

流程一致，同样需要滑动滑块开始、到达对应加油站节点登记，任务完成后归档至 **已完成** 台账，用于油耗台账线上留痕对账。

### 任务显示晚点、产生系统罚款怎么处理？

先在 **已完成** 台账查看晚点时长、异常记录。

如存在合理客观原因，例如高速封路、冷机故障、道路事故，可点击 **申诉** 上传现场异常照片凭证，等待调度审核减免罚款；无有效凭证时，罚款自动生效。

### 中转分拨到达不滑动滑块登记会有什么影响？

系统无中转到达时间记录，整趟任务可能被判定为全程晚点，并自动生成系统罚款。

所有途经分拨节点抵达后，必须完成到达滑块登记。

### 双驾车辆创建任务怎么添加随车司机？

在 **创建任务** 页面，通过 **副驾司机** 选项选择对应同车司机工号姓名。

任务记录会同步关联两名司机，工时、运输台账双方均可查看。
