# 冷链快递

## 一、适用场景

本文适用于需要操作或了解**冷链快递**业务的客户、一线网点、中心、云仓及网络货运人员。

冷链快递是中通冷链推出的 **“冷链段干线温控 + 快递段末端配送”** 混合模式产品：

- **冷链段**：负责干线运输的温度保障。
- **快递段**：由三方快递网络（中通/韵达/圆通/极兔等）负责末端时效配送。

该模式用于补充传统冷链零担无法触达末端、传统快递无法保障温控的场景。

## 四、操作步骤

### 4.1 冷链快递模式说明

| **模式** | **支持平台** | **客户订购快递公司** | **客户店铺后台展示单号** | **三方单号** |

| 冷链快递3.0 | 快手，拼多多 | 中通冷链 | 742开头单号 | 742开头单号 |

### 4.2 核心业务逻辑

冷链快递的核心逻辑为：**冷链段负责干线温度，快递段负责末端时效**。

客户下单后，货物经冷链干线运输至末中心；末中心完成冷-快交接后，由三方快递网络完成末端配送。

## 五、操作结果

完成操作后，应达到以下结果：

1. 客户已完成对应冷链快递模式的**电子面单订购**。
2. 网点已完成电子面单订购申请的**审核**。
3. 网点已完成电子面单**充值**。
4. 客户已完成冷链快递下单，并按对应模式打印可用面单：
   - **冷链快递1.0**：可通过客户打单软件打印面单，但需手写**云冷分拣码及温层**。
   - **冷链快递2.0、3.0**：需到**客户服务平台**打印标准冷链快递面单。
5. 后续货物按冷链快递流程流转：冷链干线运输至末中心，完成冷-快交接后由三方快递网络配送。

## 六、注意事项

::: danger 重点提醒
- **冷链快递1.0**：客户订购快递公司为**中通快递**，客户店铺后台展示单号为**7开头单号(除742开头)**，无三方单号。
- **冷链快递2.0**：客户订购快递公司为**中通冷链**，客户店铺后台展示单号为**8开头单号**，三方单号为**7开头单号**。
- **冷链快递3.0**：客户订购快递公司为**中通冷链**，客户店铺后台展示单号为**742开头单号**，三方单号为**742开头单号**。
:::

::: warning 注意事项
- 冷链快递1.0打印的快递单不包含**云冷分拣码及温层**，需手写补充。
- 冷链快递2.0、3.0在店铺后台或打单软件取号后，需到**客户服务平台**打印标准冷链快递面单。
- 若三方快递取号提示超区，通常与收件地址盲区或未匹配到派件网点有关，可按异常处理方案处理。
- 若提示电子面单余额不足，需网点充值并分配电子面单到具体电子面单账号。
:::

## 七、常见问题

### 7.1 如何快速区分冷链快递1.0/2.0/3.0？

可优先通过运单号区分：

| **模式** | **区分方式** |
|---|---|
| 冷链快递1.0 | 单号为**除742开头外的7开头运单** |
| 冷链快递2.0 | 运单号为**8开头**，三方单号为**7开头** |
| 冷链快递3.0 | 运单号为**742开头**，三方单号与运单号一致，均为**742开头** |

### 7.2 如何打印冷链快递面单？

可根据实际场景选择打印入口：

| **打印入口** | **说明** |
|---|---|
| 客户服务平台 | 支持客户/网点操作打印**冷链快递标准面单** |
| 鲸天系统 | **寄件运单管理**支持打印**冷链快递标准面单** |
| 其他方式 | 冷链快递1.0可通过客户打单软件打印面单，但需要手写**云冷流向码** |

### 7.3 常见异常如何处理？

| 序号 | ❌ 异常现象 / 报错提示 | 🔍 常见原因 | 🛠️ 解决方案 |
|------|-------------------------------|-----------------|--------------------|
| 1 | 调用三方快递取号报超区 | 1，收件地址在盲区，未匹配到派件网点 | 换快递公司发货 |
| 2 | 调用三方快递取号报电子面单余额不足 | 1，三方快递电子面单余额不足 | 网点充值分配电子面单到具体的电子面单账号 |
| 3 | 目的中心匹配失败，请检查路由配置！ | 冷链快递路由未配置 | 通知总部运营配置冷链快递路由 |
