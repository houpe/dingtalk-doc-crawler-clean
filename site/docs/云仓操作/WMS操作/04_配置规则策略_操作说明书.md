# 配置规则策略

## 一、适用场景

本文档用于指导WMS系统上线后配置核心业务规则策略，解决以下问题：

- 出入库作业需要经历哪些操作环节（复核、打包、称重等）。
- 操作时需遵守哪些验证规则（库位、货品编码、数量校验等）。
- 如何按仓库、货主、单据类型差异化配置入库上架、出库库存分配、订单组波、承运方案等策略。
- 常见配置异常（如出库包裹无法更新状态）的排查与修复。

## 二、前置条件

- **账号与权限**：需拥有总部管理员权限。若无权限，请联系系统管理员开通。
- **基础数据已创建**：已完成仓库、货主、员工、库位、SKU的创建。
- **业务需求明确**：了解货主的出入库流程要求（是否需要复核/打包/称重）、承运商信息、定位策略偏好（空库位优先 / 补满库位优先）等。

## 三、操作入口

请以系统实际菜单路径为准。主要配置模块路径如下：

| 配置类型 | 菜单路径 |
|---------|---------|
| 作业流程 | 策略 → 作业流程 |
| 操作配置 | 配置 → 操作配置 |
| 仓库货主参数 | 配置 → 仓库货主配置 |
| 策略规则（定位、分配、波次等） | 配置 → 策略 |

## 四、操作步骤

### 4.1 场景一：配置作业流程与操作配置

**步骤1：配置作业流程**

1. 进入菜单 **策略 → 作业流程**。
2. 按「仓库 + 货主 + 单据类型 + 流程类型」组合创建作业流程节点。
   - 若不配置，出库包裹无法更新状态，且无法打开出库包裹详情页。
   - 示例：若打包仅为计件（实际打包在复核环节完成），可关闭打包节点。
3. 点击 **保存**。

![](./images/01_创建仓库、货主、员工_操作说明书_10_e94b96dc.png)

**步骤2：配置操作配置**

1. 进入菜单 **配置 → 操作配置**。
2. 定义仓库货主在满足条件时需要遵循的操作验证规则（如是否校验库位、货品编码、数量等）。

![](./images/01_创建仓库、货主、员工_操作说明书_11_e213f0ae.png)

**步骤3：将复核、打包、称重操作配置为“无需验证”**

- 在操作配置页面，找到对应操作节点，设置为 **无需验证**。

![](./images/01_创建仓库、货主、员工_操作说明书_12_4b5dcb0d.png)

**步骤4：将相关校验项配置为“否”**

- 根据业务需要，关闭不需要的校验项（如强制扫描条码等）。

![](./images/01_创建仓库、货主、员工_操作说明书_13_2ccb6f1a.png)

![](./images/01_创建仓库、货主、员工_操作说明书_14_457f7d24.png)

### 4.3 场景三：配置策略规则

**步骤1：进入菜单 **配置 → 策略**。

**步骤2：按需创建/编辑以下21项策略规则（核心策略详解在后）**

| 策略名称 | 说明 |
|---------|------|
| 作业流程 | 不同货主+流程类型+单据类型的作业流程 |
| 定位规则 | 上架定位规则：按区域、货主、货品等筛选库位，按优先级定位 |
| 拣货任务拆分规则 | B2B波次使用，按维度拆分创建任务单 |
| 分配规则 | 出库库存分配规则（库位分配原则、属性分配原则） |
| 分配条件 | 将分配规则关联到具体业务场景 |
| 波次规则 | 订单组波规则（波次类型、分组、分配规则、打单模式等） |
| 集货规则 | 集货区选择规则，应用于B2B波次 |
| 包装方案 | 为包裹推荐和指定包装方式的配置 |
| B2B分配条件 | B2B订单的分配规则 |
| B2B波次规则 | B2B出库的波次规则 |
| 拣货任务拆分规则(2B) | 将分配请求或包裹明细分组合并创建拣货任务 |
| 赠品规则 | 货主、销售平台与赠品的关系 |
| 拆包规则 | 自动拆包规则，配合组合货品使用 |
| 序列号管理 | SKU如何管理序列号 |
| 序列号规则 | 如何截取保存序列号 |
| 指定包装方案 | 为包裹指定包装方案的规则 |
| 盘点计划 | 周期盘点计划，可自动产生盘点单 |
| 条码截取规则 | 外来条码过长时截取规则 |
| 打印设置 | 业务流程打印模板与规则的绑定 |
| 承运方案 | 为包裹分配承运商+网点+产品类型+增值服务 |
| SKU接口更新规则 | OMS再次推送SKU时允许更新的字段规则 |

**步骤3：定位策略详解**

1. 上架定位规则：根据基本单位、货主、库存类型筛选库位范围，按定位原则和上架策略排序，循环确定定位库位与数量。当剩余待定位数量不足第一优先级基本单位时，顺次选择第二优先级，直至全部收货数量定位完成。
   - **定位原则**：绑定区域优先——绑定货主/货品的库位优先分配。
   - **上架策略 - 空库位优先**：就近找符合条件的空库位。
   - **上架策略 - 补满库位优先**：在符合条件库位范围内找到未装满库位填满，不会拆分基本单位（如基本单位100件，库位剩余容量80件则跳过）。

2. 上架定位条件：将货主+单据类型与定位规则关联，决定使用哪条定位规则。

## 五、操作结果

- 配置完成后，对应规则立即生效。
- 出入库作业将按配置的作业流程、操作验证规则、定位规则、分配规则、波次规则等自动执行。
- 可在相应页面查看已配置的规则列表，测试验证是否正常生效。

## 六、注意事项

::: warning 注意事项
- 作业流程未配置或配置不完整，会导致出库包裹无法更新状态、无法打开详情页。
- 策略需关联到对应的货主+单据类型（如定位条件、分配条件）才能生效；未关联则不生效。
- 策略优先级排序会影响执行顺序，配置时注意检查是否被其他策略覆盖。
- 补满库位优先策略下，不会拆分基本单位，若库位剩余容量小于一个基本单位则跳过该库位。
- 仓库货主参数较多，建议逐项核对业务需求，不要遗漏关键开关（如重量验证、自动拆包等）。
- 修改配置后建议在小范围测试通过后再全面应用。
:::

::: danger 重点提醒
- 配置操作配置时，若将复核、打包、称重等节点设为“无需验证”，则对应环节不会校验；请确保业务允许的情况再操作。
- 序列号强校验开启后，如果入库单含计划序列号，必须扫描与计划一致的序列号，否则报错。
- 出库包裹无法更新状态、详情页打不开时，优先检查作业流程是否配置完整。
:::

## 七、常见问题

### 7.1 Q1：为什么配了策略但没生效？

**A**：请检查以下几点：
1. 策略是否已关联到对应的货主+单据类型（如定位条件、分配条件）。
2. 策略优先级排序是否正确。
3. 是否被其他更高优先级的策略覆盖。

### 7.2 Q2：定位策略“补满库位优先”和“空库位优先”如何选择？

**A**：
- **空库位优先**：适用于希望货物分散存放、提高库位利用率的场景。
- **补满库位优先**：适用于希望集中存放、方便管理的场景。注意该策略不会拆分基本单位（如基本单位100件，库位剩余容量80件则跳过）。

### 7.3 Q3：出库包裹无法更新状态或无法打开详情页怎么办？

**A**：进入 **策略 → 作业流程**，检查是否配置了对应仓库+货主+单据类型+流程类型的作业流程。若缺失，补充配置即可。
