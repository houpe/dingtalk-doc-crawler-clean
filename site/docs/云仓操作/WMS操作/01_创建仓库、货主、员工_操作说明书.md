# 创建仓库、货主、员工

## 一、适用场景

本文适用于 **WMS 系统初始化基础配置**，面向 **总部管理员**，用于完成以下基础数据配置：

1. 创建组织架构数据：**公司、部门、仓库**。
2. 创建员工并分配权限：包含 **总部角色/总部员工**、**仓库管理员**、**仓库员工**。
3. 创建货主并配置参数：包含 **入驻仓库**、**单据类型配置**、**货主配置**、**仓库货主配置**、**取号店铺配置**。

这些基础数据是 WMS 系统运行的基础，后续的库位管理、出入库作业、策略配置等都依赖这些数据。

## 二、前置条件

- **账号与权限要求**：需要拥有 **总部管理员（租户级别）权限**。如无权限，请联系系统管理员开通。
- **系统租户**：已开通 WMS 系统租户账号。
- **基础信息准备**：
  - 已明确 **公司、部门、仓库** 的命名和组织关系。
  - 已确定仓库的 **合作性质**：**自营** 或 **加盟**。
  - 已明确员工、角色、货主等基础资料信息。

## 三、操作入口

- 创建仓库入口：**系统管理 -> 基础资料 -> 公司/部门/仓库**
- 创建总部员工与总部权限入口：**权限管理**
- 创建仓库员工与仓库权限入口：**仓库权限**
- 创建货主入口：**系统管理 -> 基础资料 -> 货主管理**

## 四、名词说明

- **公司**：可理解为仓库结算主体，仓库需要关联公司。
- **部门**：仓库需要关联部门，一个部门仅关联一个仓库。员工也需要关联部门，系统按部门分权限。一个部门可理解为一个仓库。
- **仓库**：物理仓库在系统中的数字化映射。创建仓库时需要选择合作性质：**自营** 或 **加盟**。
- **总部员工**：租户级别员工，拥有全局管理权限，使用 **权限管理** 菜单下的功能。
- **仓库员工**：仓库级别员工，由仓库管理员创建和管理，使用 **仓库权限** 菜单下的功能。
- **仓库管理员**：仓库管理者，由总部创建并分配权限规则（最大权限集合），可继续为仓库员工分配权限。
- **权限规则**：菜单权限集合，用于统一赋予仓库管理员，定义仓库管理员的最大权限集合。
- **货主**：货物归属方/货品所有者。货主可入驻多个仓库，成为多仓货主。
- **标准单据类型**：WMS 的业务单据类型与中通云仓结算系统标准类型的映射关系。不需要时可不配置。

## 五、操作步骤

### 5.1 创建仓库

1. 进入 **系统管理 -> 基础资料 -> 公司/部门/仓库**。

2. 新建公司。

   公司可理解为仓库结算主体，仓库需要关联公司。

   ![](./images/01_创建仓库、货主、员工_操作说明书_10_e94b96dc.png)

3. 新建部门。

   仓库需要关联部门，部门仅关联一个仓库。员工也需要关联部门，员工仅属于一个部门。系统按部门分配权限。

   ::: tip 说明
   一个部门可以理解为一个仓库。员工本身仅属于一个仓库，但可通过权限分配看到多个仓库的数据。
   :::

   ![](./images/01_创建仓库、货主、员工_操作说明书_11_e213f0ae.png)

4. 新建仓库。

   为员工分配部门权限后，员工即可拥有该部门关联仓库的数据权限。

   ![](./images/01_创建仓库、货主、员工_操作说明书_12_4b5dcb0d.png)

   ::: warning 注意事项
   仓库合作性质包括 **自营**、**加盟**，不同合作性质在系统中会有差异。例如：**加盟仓库位可不校验库位编码规则，可使用自定义库位编码**。
   :::

5. 修改仓库级别参数。

   仓库级别参数修改后，**全仓生效**。

   ![](./images/01_创建仓库、货主、员工_操作说明书_13_2ccb6f1a.png)

### 5.2 创建员工与分配权限

1. 根据员工类型进入对应菜单：

   - 总部员工使用：**权限管理**
   - 仓库员工使用：**仓库权限**

   ![](./images/01_创建仓库、货主、员工_操作说明书_14_457f7d24.png)

2. 了解权限管理架构。

   - **租户级别（仅总部员工使用）**：**权限管理 > 部门与员工 > 角色管理 > 仓库货主权限 > 仓库管理员 > 权限规则 > 权限下放回收**
   - **仓库级别（仅仓库员工使用）**：**仓库权限 > 仓库货主权限 > 员工管理 > 角色管理**

3. 创建总部角色。

   新建总部角色，或新建部门下的角色。

   ![](./images/01_创建仓库、货主、员工_操作说明书_15_7567dfc0.png)

4. 创建总部员工。

   新建总部员工，并绑定 **角色**、**部门**。绑定部门即绑定了仓库，员工属于该仓库。

   ![](./images/01_创建仓库、货主、员工_操作说明书_16_ef5e73cc.png)

5. 为员工分配仓库权限。

   分配时选中的是 **部门**，即员工可查看这些部门对应的仓库。

   ![](./images/01_创建仓库、货主、员工_操作说明书_17_9cc56e8f.png)

6. 分配仓库货主权限。

   在 **仓库货主权限** 中，为员工分配该部门仓库下的货主权限。

   ![](./images/01_创建仓库、货主、员工_操作说明书_18_42ee3433.png)

7. 创建权限规则。

   新建 **权限规则**。权限规则是菜单权限集合，用于统一赋予仓库管理员，也是仓库管理员的最大权限集合，管理员可继续分配给仓库员工。

8. 创建仓库管理员。

   创建仓库管理员职位后，为该职位添加员工，作为仓库管理员。

   ![](./images/01_创建仓库、货主、员工_操作说明书_19_730c34e3.png)

9. 仓库管理员登录账号后，可在 **仓库权限** 菜单下创建仓库级别员工、角色，并分配货主权限。

   仓库员工使用以下菜单：

   - **仓库权限-仓库货主权限**
   - **仓库权限-员工管理**
   - **仓库权限-角色管理**

   ::: warning 注意事项
   总部员工才使用 **权限管理** 下的菜单。
   :::

   ![](./images/01_创建仓库、货主、员工_操作说明书_1_68925fe0.png)

### 5.3 创建货主

1. 进入 **系统管理 -> 基础资料 -> 货主管理**。

2. 添加货主。

   配置货主基础信息，并选择 **入驻仓库**。货主入驻多个仓库时，即为多仓货主。

   ![](./images/01_创建仓库、货主、员工_操作说明书_20_e59e47f8.png)

   ::: danger 重点提醒
   **联系人** 可作为取号面单发件人，**必须配置**，否则会导致取号失败。
   :::

3. 配置单据类型。

   **单据类型配置** 用于维护 WMS 的业务单据类型与标准单据类型的关系。标准单据类型即中通云仓结算系统的标准类型，不需要时可不配置。

   ![](./images/01_创建仓库、货主、员工_操作说明书_2_5e45b2a3.png)

4. 配置货主维度参数。

   **货主配置** 为货主维度参数，不区分仓库，对该货主全部入驻仓库生效。

   ![](./images/01_创建仓库、货主、员工_操作说明书_3_925b7af9.png)

5. 配置仓库货主参数。

   **仓库货主配置** 是仓库下的货主参数，仅在货主入驻的该仓生效。请根据业务需要为仓库货主配置相关业务参数。部分参数需要特定配置，可参考下方截图。

   ![](./images/01_创建仓库、货主、员工_操作说明书_4_211d769e.png)

   ![](./images/01_创建仓库、货主、员工_操作说明书_5_1bc99e39.png)

   ![](./images/01_创建仓库、货主、员工_操作说明书_6_464c1e9c.png)

   ![](./images/01_创建仓库、货主、员工_操作说明书_7_1a9ea66b.png)

   ![](./images/01_创建仓库、货主、员工_操作说明书_8_11d55e9c.png)

6. 配置取号店铺。

   为货主创建取号店铺，或授权该货主使用其他取号店铺，用于云取号、打印面单。

   ::: tip 说明
   现在不需要为货主开通店铺。当前仓库不配置货主店铺、货主的承运商配置时，全部使用 **仓库货主** 配置。
   :::

   ![](./images/01_创建仓库、货主、员工_操作说明书_9_c1a2da40.png)

## 六、操作结果

完成以上配置后，系统应具备以下基础数据和权限结果：

1. 已创建 **公司、部门、仓库**，且仓库已关联对应公司和部门。
2. 已配置仓库级别参数，相关参数在对应仓库范围内生效。
3. 已创建总部员工、角色，并完成 **仓库权限**、**仓库货主权限** 分配。
4. 已创建仓库管理员，仓库管理员可进入 **仓库权限** 菜单管理仓库员工、角色和货主权限。
5. 已创建货主，并完成入驻仓库、单据类型、货主参数、仓库货主参数及取号店铺相关配置。

## 七、注意事项

::: danger 重点提醒
- **联系人** 可作为取号面单发件人，货主联系人 **必须配置**，否则取号失败。
- 总部员工使用 **权限管理** 下的菜单；仓库员工使用 **仓库权限** 下的菜单。
- 员工本身仅属于一个部门（仓库），但可通过分配仓库权限查看多个仓库的数据。
:::

::: warning 注意事项
- 仓库合作性质为 **自营** 或 **加盟**，系统表现会有差异。例如：**加盟仓库位可不校验库位编码规则，可使用自定义库位编码**。
- **货主配置** 为货主维度参数，不分仓库，全部入驻仓库生效。
- **仓库货主配置** 为仓库下的货主参数，仅在货主入驻的该仓生效。
- **标准单据类型** 是中通云仓结算系统的标准类型，不需要时可不配置。
:::

## 八、常见异常与兜底方案

| **序号** | **异常现象** | **常见原因** | **解决方案** |
|----------|----------------|----------------|----------------|
| 1 | 找不到寄件人信息而取号失败 | 货主联系人未配置 | 在货主管理中补充联系人信息，联系人作为取号面单发件人 |
| 2 | 员工看不到对应仓库数据 | 未分配部门权限或仓库货主权限 | 检查【权限管理】中是否已为员工分配对应部门的仓库权限和货主权限 |
| 3 | 仓库管理员无法创建员工 | 权限规则未创建或未分配给仓库管理员 | 先在【权限管理-权限规则】创建规则，再创建管理员职位并分配 |

## 九、常见问题

### 9.1 加盟仓库和自营仓库有什么区别？

**答**：加盟仓库位可不校验库位编码规则，使用自定义库位编码。自营仓库需要遵循标准库位编码规则。其他功能基本一致。

### 9.2 一个员工可以属于多个仓库吗？

**答**：员工本身仅属于一个部门（仓库），但可以通过分配仓库权限看到多个仓库的数据。

### 9.3 货主维度和仓库货主维度参数有什么区别？

**答**：货主维度参数不分仓库，全部入驻仓库生效。仓库货主维度参数仅在货主入驻的该仓生效。
