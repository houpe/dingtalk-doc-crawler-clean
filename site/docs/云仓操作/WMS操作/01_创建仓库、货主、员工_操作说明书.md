# 创建仓库、货主、员工

## 一、适用场景

WMS 系统初始化时，需要先搭建基础组织架构数据（公司、部门、仓库、员工、货主）。这些数据是库位管理、出入库作业、策略配置的前提。

正确配置组织架构能决定：
- **权限分配方式**（按部门分权）
- **仓库数据隔离**
- **货主业务参数生效范围**

## 二、前置条件

- **账号与权限**：拥有**总部管理员（租户级别）**账号。若无权限，请联系系统管理员开通。
- **数据准备**：已开通 WMS 系统租户账号；明确公司、部门、仓库的命名和组织关系；确定仓库的合作性质（**自营** / **加盟**）。

## 三、操作入口

下方每个场景均标注了对应的系统功能路径，请以系统实际菜单为准。

## 四、操作步骤

### 4.1 场景一：创建仓库

**系统功能路径**：系统管理 → 基础资料 → 公司 / 部门 / 仓库

#### (1) 新建公司

公司是仓库的结算主体，仓库需关联公司。

![](./images/01_创建仓库、货主、员工_操作说明书_10_e94b96dc.png)

#### (2) 新建部门

仓库需关联部门，一个部门仅关联一个仓库。员工也需关联部门，员工仅属于一个部门。系统按部门分权限，一个部门可理解为一个仓库，员工可看到多个仓库但本身只属于一个部门。

![](./images/01_创建仓库、货主、员工_操作说明书_11_e213f0ae.png)

#### (3) 新建仓库

为员工分配部门权限后，员工即拥有该部门关联的仓库数据权限。

![](./images/01_创建仓库、货主、员工_操作说明书_12_4b5dcb0d.png)

::: warning 注意事项
**合作性质**（自营 / 加盟）存在差异：加盟仓库的库位可不校验编码规则，允许使用自定义库位编码。
:::

#### (4) 修改仓库级别参数

修改仓库级别参数后，全仓生效。

![](./images/01_创建仓库、货主、员工_操作说明书_13_2ccb6f1a.png)

### 4.2 场景二：创建员工与分配权限

**系统功能路径**：总部使用【权限管理】；仓库使用【仓库权限】

![](./images/01_创建仓库、货主、员工_操作说明书_14_457f7d24.png)

#### (1) 权限管理架构说明

- **租户级别（仅总部员工使用）**：权限管理 → 部门与员工 → 角色管理 → 仓库货主权限 → 仓库管理员 → 权限规则 → 权限下放回收
- **仓库级别（仅仓库员工使用）**：仓库权限 → 仓库货主权限 → 员工管理 → 角色管理

#### (2) 创建总部员工与角色

1. **新建总部角色**：在总部角色管理下，新建所属部门的角色。
   ![](./images/01_创建仓库、货主、员工_操作说明书_15_7567dfc0.png)

2. **新建总部员工**：绑定角色和部门。绑定部门即绑定了仓库，员工属于该仓库。
   ![](./images/01_创建仓库、货主、员工_操作说明书_16_ef5e73cc.png)

3. **分配仓库权限**：为员工勾选可查看的部门（即员工可以看到这些仓库的数据）。
   ![](./images/01_创建仓库、货主、员工_操作说明书_17_9cc56e8f.png)

4. **分配仓库货主权限**：为员工分配该部门仓库下可操作的货主。
   ![](./images/01_创建仓库、货主、员工_操作说明书_18_42ee3433.png)

#### (3) 创建仓库角色与员工

1. **新建权限规则**：定义菜单权限集合，用于统一赋予仓库管理员（管理员的**最大权限集合**）。
2. **创建仓库管理员职位**：为该职位添加员工，成为仓库管理员。
   ![](./images/01_创建仓库、货主、员工_操作说明书_19_730c34e3.png)

3. **仓库管理员登录后**：在【仓库权限】菜单（下方截图）下，可管理仓库级别的 **员工、角色、仓库货主权限**。
   ![](./images/01_创建仓库、货主、员工_操作说明书_1_68925fe0.png)

::: warning 注意事项
- **总部员工**才使用【权限管理】菜单；**仓库员工**使用【仓库权限】菜单。
- 仓库员工可操作的菜单为：**【仓库权限 - 仓库货主权限】**、**【仓库权限 - 员工管理】**、**【仓库权限 - 角色管理】**。
:::

### 4.3 场景三：创建货主

**系统功能路径**：系统管理 → 基础资料 → 货主管理

#### (1) 添加货主

- 选择入驻的仓库（入驻多仓即为多仓货主）。
  ![](./images/01_创建仓库、货主、员工_操作说明书_20_e59e47f8.png)

::: danger 重点提醒
**联系人**必须填写，否则取号（面单发件人）会失败。
:::

#### (2) 单据类型配置（可选）

配置 WMS 业务单据类型与中通云仓结算系统标准类型的映射关系。若不需要可跳过。

![](./images/01_创建仓库、货主、员工_操作说明书_2_5e45b2a3.png)

货主维度参数对货主**全部入驻仓库**生效。

![](./images/01_创建仓库、货主、员工_操作说明书_3_925b7af9.png)

#### (4) 仓库货主配置

参数仅在货主入驻的**该仓**生效。根据需要配置相关业务参数。

![](./images/01_创建仓库、货主、员工_操作说明书_4_211d769e.png)

![](./images/01_创建仓库、货主、员工_操作说明书_5_1bc99e39.png)

![](./images/01_创建仓库、货主、员工_操作说明书_8_11d55e9c.png)

#### (5) 取号店铺配置（可选）

为货主创建或授权取号店铺，用于云取号、打印面单。

::: tip 说明

## 五、操作结果

- **仓库创建完成**：在公司、部门、仓库列表可查看到新增记录。
- **员工创建完成**：在员工管理列表可查看到新增员工，且具备对应仓库和货主的数据权限。
- **货主创建完成**：在货主管理列表可查看到新增货主，相关参数（货主维度、仓库货主维度）已生效。

## 六、注意事项

- **合作性质差异**：加盟仓库的库位编码不强制校验规则，可使用自定义库位编码。
- **员工归属**：员工本身仅属于一个部门（仓库），但可通过仓库权限看到多个仓库的数据。
- **权限菜单区分**：总部员工使用【权限管理】；仓库员工使用【仓库权限】。
- **货主联系人**：必须填写，否则取号（面单发件人）会失败。

## 七、常见问题

### 7.1 异常处理

| 异常现象 | 常见原因 | 解决方案 |
|----------|----------|----------|
| 找不到寄件人信息导致取号失败 | 货主联系人未配置 | 在货主管理中补充联系人信息，联系人作为取号面单发件人 |
| 员工看不到对应仓库数据 | 未分配部门权限或仓库货主权限 | 在【权限管理】中为员工分配对应部门的仓库权限和货主权限 |
| 仓库管理员无法创建员工 | 权限规则未创建或未分配给仓库管理员 | 先在【权限管理 - 权限规则】创建规则，再创建管理员职位并分配 |

### 7.2 高频问答

**Q：加盟仓库和自营仓库有什么区别？**
A：加盟仓库的库位可不校验编码规则，允许自定义库位编码；自营仓库需遵循标准库位编码规则。其他功能基本一致。

**Q：一个员工可以属于多个仓库吗？**
A：员工本身仅属于一个部门（仓库），但可通过分配仓库权限查看多个仓库的数据。

**Q：货主维度和仓库货主维度参数有什么区别？**
A：货主维度参数不分仓库，对所有入驻仓库生效；仓库货主维度参数仅在货主入驻的该仓生效。
