# 一. 库内作业

■ <strong>文档基本信息</strong>

本文档覆盖WMS库内作业全流程：盘点→移库→补货→库存状态调整→加工单（组合套装/预包）→增值服务。含盘点计划操作说明、盘点开单/任务/盈亏/调整、PDA/PC移库操作、补货功能全览（含补货预警/补货任务）、库存状态日志、加工单创建运行完工、增值服务登记等完整作业链路。

## 1.1 业务场景与名词解释

### 1.1.1 业务场景

1. 盘点场景：仓库定期或临时发起的库存实物核对作业，支持明盘/盲盘模式。盘点结果自动生成盈亏单据，经确认后触发调整单据完成库存修正。
2. 移库场景：仓库内部库位间的库存转移作业。PDA直接移库支持扫描即移；PDA下架上架需借助容器中转；PC端支持批量在途创建移库计划。
3. 补货场景：拣选区库存不足时，从存储区向拣选区补充库存。支持多种触发方式（出库包裹补货分析、安全库存预警、包裹汇单批量补货、加入波次缺货判断），自动生成补货预警与补货任务。
4. 库存状态调整：对库位库存的非数量变更操作（如锁定、冻结、属性变更），操作结果记录调整单据与库存日志。
5. 加工单场景：组合套装——将多个SKU组合为新SKU增加库存；预包——预先完成商品包装组合，下游出库时直接调用预包结果。
6. 增值服务场景：非标准化的仓库附加服务，因流程个性化需线下作业、线上登记。

### 1.1.2 核心名词解释

- <strong>盘点开单</strong>：创建一次盘点任务，选择盘点范围（库区/库位/SKU）、盘点方式（明盘/盲盘）。
- <strong>盈亏单据</strong>：盘点差异（盘盈/盘亏）自动生成的差异单据，确认后产生调整单。
- <strong>调整单据</strong>：所有库存调整的统一记录入口，包括盘点调整、库位库存状态属性调整、效期属性自动调整等。
- <strong>直接移库</strong>：不借助容器，从下架库位直接扫描转移到上架库位的移库方式。
- <strong>移库下架上架</strong>：需借助容器周转：下架到容器→容器转移→从容器上架到目标库位。
- <strong>补货分析</strong>：对比订单需求数量与拣选区/存储区可用数量，生成补货建议。
- <strong>补货预警</strong>：根据订单池、安全库存分析自动生成的补货提醒，可据此创建补货任务。
- <strong>加工单</strong>：将多品组合为新品的作业单据，运行后生成拣货任务，完工确认后新品上架。
- <strong>预包</strong>：预先完成商品包装组合的加工方式，使下游出库时直接调用预包结果。

## 1.2 前置准备与环境配置

- <strong>权限要求</strong>：需有仓库管理员权限，完成货主档案配置（如盘点调整权限开关）。
- <strong>系统配置</strong>：货主档案中开启盘点调整允许；组合货品需在【组合货品】页面预先维护；安全库存需在库位配置中设置。
- <strong>设备要求</strong>：PC端操作（盘点开单/任务确认/调整单据/加工单管理/补货预警）；PDA端操作（直接移库/移库下架上架/补货任务执行）。
- <strong>容器准备</strong>：移库下架上架和补货任务需使用容器中转，提前编码容器并在系统中维护。

### 1.3.2 盘点流程详解

#### 1.3.2.1 盘点开单

盘点开单是创建盘点任务的入口。操作时需选择：盘点范围（库区/库位/SKU/全仓）、盘点方式（明盘：显示系统库存数量；盲盘：不显示系统库存数量）。

![](./images/01_创建仓库、货主、员工_操作说明书_13_2ccb6f1a.png)

#### 1.3.2.2 盘点任务

盘点开单后系统会自动生成盘点任务。在盘点任务页面可进行如下操作：

- <strong>分配盘点人</strong>：必须分配盘点人账号，否则无法提交实盘数量。可分配给指定用户，未分配则可在PDA自由领取。

![](./images/01_创建仓库、货主、员工_操作说明书_14_457f7d24.png)

- <strong>编辑录入实盘数量</strong>：点击编辑，录入实际盘点数量，保存即可。

![](./images/01_创建仓库、货主、员工_操作说明书_15_7567dfc0.png)

![](./images/01_创建仓库、货主、员工_操作说明书_16_ef5e73cc.png)

- <strong>打印盘点单</strong>：如需纸质盘点单，点击打印按钮即可打印纸质盘点单用于线下作业。

![](./images/01_创建仓库、货主、员工_操作说明书_17_9cc56e8f.png)

#### 1.3.2.3 盈亏单据

对于盘点结果，如果存在盘盈或盘亏差异，系统会将差异自动创建盈亏单据。

- <strong>点调整</strong>：产生调整单据，用于后续库存修正。
- <strong>复盘触发</strong>：如果同一SKU存在多条不同属性的库存，系统会要求复盘。复盘会生成新的盘点任务，需要重新录入实盘数量。

![](./images/01_创建仓库、货主、员工_操作说明书_18_42ee3433.png)

#### 1.3.2.4 调整单据

对于已确认的盈亏单据，系统自动产生调整单据。确认调整单据后，会正式调整库存数量。

![](./images/01_创建仓库、货主、员工_操作说明书_19_730c34e3.png)

重要补充：所有库存的调整都会进入调整单据，包括盘点调整、库位库存状态属性调整、依据SKU档案效期属性的自动调整等。

### 1.3.3 PDA直接移库

2. <strong>系统功能路径</strong>：PDA -> 直接移库

直接移库是最便捷的库内库存转移方式，无需借助容器中转。适用于单个库位间的快速库存调整。

3. <strong>操作步骤</strong>：扫描下架库位编码（或手动选择）→ 扫描/输入商品条码 → 选择上架库位 → 确认完成移库。

![](./images/01_创建仓库、货主、员工_操作说明书_4_211d769e.png)

### 1.3.4 PDA移库下架 & 移库上架

4. <strong>系统功能路径</strong>：PDA -> 移库下架 / 移库上架

与直接移库不同，此方式需借助容器作为中转。适用于跨库区移库、批量移库或需要作业记录的场景。

5. <strong>移库下架操作</strong>：领取/创建移库任务 → 扫描下架库位和商品 → 需下架到指定容器（扫描容器编码）→ 完成下架，商品进入容器。
6. <strong>移库上架操作</strong>：扫描容器编码 → 确认容器中的商品 → 扫描上架库位 → 确认数量 → 完成上架，容器释放。

### 1.3.5 PC移库

7. <strong>系统功能路径</strong>：库内 -> PC-移库

PC端支持批量创建移库计划，适用于大量库存的整体转移。在PC端创建移库单据后，可在PDA端领取并执行。

![](./images/01_创建仓库、货主、员工_操作说明书_7_1a9ea66b.png)

### 1.3.6 补货功能全览

补货功能是WMS库内作业的重要组成部分，当拣选区库存不足以满足出库需求时，需从存储区向拣选区补充库存。系统提供多种补货触发方式和操作入口，以下为补货功能全览表：

| <strong>补货触发场景</strong> | <strong>操作入口/页面</strong> | <strong>功能说明</strong> |
|----------------------|-----------------------|----------------|
| 出库-缺货提醒、补货 | PC-出库包裹-补货分析 | 根据单据需求、拣选、存储区域库存，生成补货建议。 |
| 出库-缺货提醒、补货 | PC-包裹汇单-补货分析 | 对待加入波次包裹批量补货分析，自动生成补货预警 |
| 出库-缺货提醒、补货 | PC-安全库存 | 设置拣选区库位库存的安全库存，自动分析与指导补货，当触及预警线，自动生成从存储区到拣选区的补货预警 |
| 出库-缺货提醒、补货 | PC-【出库包裹】【包裹汇单】加入波次 | 加入波次判断缺货，自动生成补货预警 |
| 出库-缺货提醒、补货 | PC- 补货预警 | 查看补货预警，批量生成补货任务，PDA- 补货任务操作补货下架然后补货上架 |
| 出库-缺货提醒、补货 | PC-补货任务 | 仅查看或取消补货任务，前往 PDA-补货任务执行 |
| 出库-缺货提醒、补货 | PDA-补货预警 | PDA 自动根据订单池、安全库存分析，自动生成补货预警，依据补货预警创建补货任务 |
| 出库-缺货提醒、补货 | PDA-补货任务 | PDA领取并完成补货任务 |

### 1.3.10 加工单组合套装

11. <strong>系统功能路径</strong>：库内 -> 加工单 -> 组合套装

<strong>功能说明</strong>：将多品（多个SKU）组合为一个新品（新SKU），通过加工单完成组合后，增加新品库存，同时扣减原材料多品库存。

12. <strong>系统功能路径</strong>：库内 -> 加工单 -> 预包

预包是预先完成商品包装组合的加工方式。操作前需在【组合货品】中新增预包方案，定义预包包含的SKU及数量。

## 1.4 常见异常与兜底方案

| <strong>序号</strong> | <strong>异常现象</strong> | <strong>常见原因</strong> | <strong>解决方案</strong> |
|----------|----------------|----------------|----------------|
| 1 | 盘点不允许调整 | 货主档案中未开启调整权限 | 前往货主档案配置，开启盘点调整开关 |
| 2 | 无法提交实盘数量 | 未分配盘点人账号 | 在盘点任务页面分配盘点人后再执行操作 |
| 3 | 复盘要求触发 | 同一SKU存在多条不同属性库存（批次/效期不同） | 按系统提示完成复盘操作，会生成新的盘点任务 |
| 4 | 移库容器无法释放 | 移库上架未完成或容器中有未确认商品 | 检查容器内所有商品是否已完成上架确认 |
| 5 | 补货任务生成失败 | 存储区库存不足以补充；补货预警未触发 | 确认存储区库存数量；检查安全库存设置是否合理 |
| 6 | 加工单运行失败 | 原材料库存不足；组合货品方案未维护 | 检查原材料库存；在【组合货品】页面确认方案已维护 |

## 1.5 高频常见问题（FAQ）

14. <strong>Q1：盘点开单后可以修改盘点范围吗？</strong>

<strong>A</strong>：盘点开单后一般不建议修改范围。如确实需要调整，可取消当前盘点计划，重新创建盘点单并选择正确的范围。

15. <strong>Q2：直接移库和移库下架上架有什么区别？</strong>

<strong>A</strong>：直接移库无需容器中转，适合单库位快速移库；移库下架上架需要借助容器中转，适合跨库区移库、批量移库或需要作业记录的正式移库场景。

16. <strong>Q3：补货分析什么时候会触发？</strong>

<strong>A</strong>：补货分析在多个环节自动触发：出库包裹页面按包裹逐单分析、包裹汇单页面对待加入波次包裹批量分析、出库包裹/包裹汇单加入波次时判断缺货、以及安全库存触及预警线时。可在补货预警页面统一查看和生成补货任务。

17. <strong>Q4：盘点产生亏损后库存何时修正？</strong>

<strong>A</strong>：盘点差异生成盈亏单据 → 点击调整产生调整单据 → 确认调整单据后 → 库存即时修正。整个链路可追溯，所有调整记录都会进入调整单据统一管理。

18. <strong>Q5：加工单（组合套装）与预包加工有什么区别？</strong>

<strong>A</strong>：组合套装是正式加工流程，将多品组合为新SKU，产生新品库存。预包加工是预先包装作业，为下游出库做备货准备，出库时可直接调用预包结果减少包装耗时。两者操作流程相同，区别在于业务目的和成品用途。

19. <strong>Q6：增值服务单如何操作？</strong>

<strong>A</strong>：增值服务因需求个性化，目前以线下作业为主。在系统端创建增值服务单进行登记，记录服务内容、工时、耗材等信息，便于后续结算和追溯。具体操作流程根据服务类型单独设计。

20. <strong>Q7：库位库存调整后在哪里查看变更记录？</strong>

<strong>A</strong>：所有库存调整（包括盘点调整、状态属性调整、效期自动调整等）均进入【调整单据】统一管理。确认后产生库存日志，可追溯到每次变更的时间、操作人、变更前后状态。
