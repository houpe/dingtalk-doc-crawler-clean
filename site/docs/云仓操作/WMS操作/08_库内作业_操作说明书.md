# 库内作业

## 一、适用场景
本文操作手册适用于以下库内作业场景：
1. 盘点：仓库定期或临时发起的库存实物核对作业，支持明盘/盲盘模式，自动生成盈亏单据并完成库存修正。
2. 移库：仓库内部库位间的库存转移作业，支持PDA直接移库（无需容器）和通过容器中转的移库下架上架。
3. 补货：拣选区库存不足时，从存储区向拣选区补充库存，支持多种触发方式（出库包裹补货分析、安全库存预警、包裹汇单批量补货、加入波次缺货判断）。
4. 库存状态调整：对库位库存的非数量变更操作（如锁定、冻结、属性变更），操作结果记录在调整单据与库存日志。
5. 加工单：组合套装（将多个SKU组合为新SKU增加库存）和预包（预先完成商品包装组合，下游出库时直接调用预包结果）。
6. 增值服务：非标准化的仓库附加服务，需线下作业、线上登记。

## 二、前置条件
| 项目 | 要求 |
|------|------|
| 权限 | 需有**仓库管理员**权限，完成货主档案配置（如盘点调整权限开关） |
| 系统配置 | 货主档案中开启**盘点调整**允许；组合货品需在【组合货品】页面预先维护；安全库存需在库位配置中设置 |
| 设备 | PC端（盘点开单/任务确认/调整单据/加工单管理/补货预警）；PDA端（直接移库/移库下架上架/补货任务执行） |
| 容器 | 移库下架上架和补货任务需使用容器中转，提前编码并维护到系统中 |

## 三、操作入口
各功能对应的系统菜单路径如下：

| 功能 | 系统路径 |
|------|----------|
| 盘点计划（开单/任务/盈亏/调整） | **库内 -> 盘点计划** |
| PDA直接移库 | **PDA -> 直接移库** |
| PDA移库下架 & 移库上架 | **PDA -> 移库下架 / 移库上架** |
| PC端移库（批量创建移库计划） | **库内 -> PC-移库** |
| 补货预警 | **库内 -> 补货预警** |
| 补货任务 | **库内 -> 补货任务** |
| 库存状态调整 | **库内 -> 库存状态调整** |
| 加工单-组合套装 | **库内 -> 加工单 -> 组合套装** |
| 加工单-预包 | **库内 -> 加工单 -> 预包** |

## 四、操作步骤

### 4.1 盘点流程

#### (1) 盘点开单
1. 进入 **库内 -> 盘点计划**，点击 **新建盘点单**。
2. 选择 **盘点范围**（库区/库位/SKU/全仓）。
3. 选择 **盘点方式**（明盘：显示系统库存数量；盲盘：不显示系统库存数量）。
4. 提交生成盘点任务。

![](./images/01_创建仓库、货主、员工_操作说明书_10_e94b96dc.png)

#### (2) 盘点任务
- **分配盘点人**：必须分配盘点人账号，否则无法提交实盘数量。可分配给指定用户；未分配则PDA上可自由领取。
- **录入实盘数量**：点击 **编辑**，录入实际盘点数量，保存。
- **打印盘点单**：如需纸质盘点单，点击 **打印** 按钮。

![](./images/01_创建仓库、货主、员工_操作说明书_11_e213f0ae.png)

![](./images/01_创建仓库、货主、员工_操作说明书_12_4b5dcb0d.png)

![](./images/01_创建仓库、货主、员工_操作说明书_13_2ccb6f1a.png)

![](./images/01_创建仓库、货主、员工_操作说明书_14_457f7d24.png)

#### (3) 盈亏单据
系统自动对比实盘数量与系统库存，生成盈亏单据（盘盈/盘亏）。
- **点调整**：产生 **调整单据**，用于后续库存修正。
- **复盘触发**：如果同一SKU存在多条不同属性的库存（如不同批次/效期），系统会要求 **复盘**，自动生成新的盘点任务，需重新录入实盘数量。

![](./images/01_创建仓库、货主、员工_操作说明书_15_7567dfc0.png)

#### (4) 调整单据
已确认的盈亏单据自动产生调整单据。确认调整单据后，库存数量即时修正。所有库存调整（盘点调整、库位库存状态属性调整、依据SKU档案效期属性的自动调整等）都会进入调整单据统一管理。

### 4.2 移库操作

#### (1) PDA直接移库
1. 进入 **PDA -> 直接移库**。
2. **扫描下架库位编码**（或手动选择）。
3. **扫描/输入商品条码**。
4. **选择上架库位**。
5. 确认完成移库。

![](./images/01_创建仓库、货主、员工_操作说明书_20_e59e47f8.png)

#### (2) PDA移库下架 & 移库上架
- **移库下架**：领取/创建移库任务 → 扫描下架库位和商品 → 扫描容器编码（需下架到指定容器） → 完成下架。
- **移库上架**：扫描容器编码 → 确认容器中的商品 → 扫描上架库位 → 确认数量 → 完成上架，容器释放。

![](./images/01_创建仓库、货主、员工_操作说明书_2_5e45b2a3.png)

PDA移库补充截图：

![](./images/01_创建仓库、货主、员工_操作说明书_3_925b7af9.png)

#### (3) PC移库
1. 进入 **库内 -> PC-移库**。
2. 批量选择需要转移的库存明细（来源库位、目标库位、SKU、数量）。
3. 创建移库单据。生成的移库计划可在PDA端领取并执行。

![](./images/01_创建仓库、货主、员工_操作说明书_4_211d769e.png)

### 4.3 补货操作
补货功能全览表：

| 补货触发场景 | 操作入口/页面 | 功能说明 |
|-------------|--------------|----------|
| 出库-缺货提醒、补货 | PC-出库包裹-补货分析 | 根据单据需求、拣选、存储区域库存，生成补货建议 |
| 出库-缺货提醒、补货 | PC-包裹汇单-补货分析 | 对待加入波次包裹批量补货分析，自动生成补货预警 |
| 出库-缺货提醒、补货 | PC-安全库存 | 设置拣选区库位库存的安全库存，自动分析与指导补货，当触及预警线，自动生成从存储区到拣选区的补货预警 |
| 出库-缺货提醒、补货 | PC-【出库包裹】【包裹汇单】加入波次 | 加入波次时判断缺货，自动生成补货预警 |
| 出库-缺货提醒、补货 | PC- 补货预警 | 查看补货预警，批量生成补货任务，PDA- 补货任务操作补货下架然后补货上架 |
| 出库-缺货提醒、补货 | PC-补货任务 | 仅查看或取消补货任务，前往 PDA-补货任务执行 |
| 出库-缺货提醒、补货 | PDA-补货预警 | PDA自动根据订单池、安全库存分析，自动生成补货预警，依据补货预警创建补货任务 |
| 出库-缺货提醒、补货 | PDA-补货任务 | PDA领取并完成补货任务 |

#### (1) 补货预警
1. 进入 **库内 -> 补货预警**。
2. 查看系统自动生成的补货建议（SKU、建议补货数量、来源库位、目标库位）。
3. 勾选需要执行的预警项，点击 **生成补货任务**。

![](./images/01_创建仓库、货主、员工_操作说明书_6_464c1e9c.png)

#### (2) 补货任务
1. 补货任务生成后，在 **PC-补货任务** 页面可查看任务状态、操作人员、完成时间。
2. 作业人员在 **PDA-补货任务** 领取任务，执行 **补货下架**（将存储区商品下架到容器）→ **补货上架**（从容器上架到拣选区目标库位）。上架完成后系统自动释放容器。

### 4.4 库存状态调整
1. 进入 **库内 -> 库存状态调整**。
2. 选择需要调整的库存记录（库位、SKU、批次等）。
3. 选择调整类型（锁定/解锁、冻结/解冻、批次属性修正等）。
4. 输入调整原因（必填），确认提交。
5. 系统生成调整单据，确认后库存状态即时变更，并在库存日志中记录完整变更链条。

### 4.5 加工单组合套装
::: danger 前置准备
操作前需在 **【组合货品】** 页面维护组合方案，定义原材料SKU与成品SKU的对应关系及用量。
:::

1. 进入 **库内 -> 加工单 -> 组合套装**，点击 **新建加工单**。
2. 选择已维护的组合货品方案，填写加工数量。
3. 点击 **运行加工单**，选择库存分配规则（如先进先出、按批次分配）。
4. 确定并运行成功后，打印拣货单。
5. 线下按拣货单将原材料组合为成品。
6. 在加工单页面点击 **完工确认**，系统自动扣减原材料库存、增加成品库存并完成上架。

可在 **库内-加工拣货** 页面查看拣货详情，追踪物料下架和完工进度。

### 4.6 加工单预包
::: danger 前置准备
操作前需在 **【组合货品】** 中新增预包方案，定义预包包含的SKU及数量。

1. 进入 **库内 -> 加工单 -> 预包**，点击 **新建预包加工单**。
2. 选择预包方案，填写加工数量。

4. 确定并运行成功后打印拣货单。
5. 线下完成预包包装。
6. 完工确认，预包成品上架（系统增加预包库存）。

预包下架操作：需将原材料移库下架到预包区，必须有规定的下架数量，系统会校验。

## 五、操作结果
- **盘点**：完成后库存数量与实际一致，调整单据可追溯。
- **移库**：库存从源库位转移到目标库位，库存状态更新。
- **补货**：拣选区库存补充完毕，补货任务完结，容器释放。
- **库存状态调整**：库存属性（锁定/冻结/批次）变更，并在调整单据和库存日志中记录。
- **加工单（组合套装）**：原材料库存扣减，成品库存增加并上架。
- **加工单（预包）**：预包库存增加，下游出库可直接调用。
- **增值服务**：系统端登记完成，服务记录可追溯。

## 六、注意事项
1. **权限与配置**：
   - 盘点调整需货主档案开启“盘点调整允许”。
   - 组合货品需先在【组合货品】页面维护方案。
   - 安全库存需在库位配置中设置。
2. **盘点**：
   - 盘点开单后一般不建议修改范围；如需调整，取消原计划重新创建。
   - 盘点任务必须分配盘点人，否则无法提交实盘数量。
   - 同一SKU多条不同属性库存会触发复盘，需重新录入。
3. **移库**：
   - 移库下架上架必须使用容器，且容器需提前在系统维护。
   - 容器内所有商品上架完成后方可释放容器。
4. **补货**：
   - 补货任务生成后需用容器中转，补货上架完成自动释放。
   - 若存储区库存不足，补货任务可能生成失败，需检查库存或调整安全库存。
5. **加工单**：
   - 运行加工单前确保原材料库存充足，否则运行失败。
