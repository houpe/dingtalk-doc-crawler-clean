# C出库作业

## 一、适用场景

本文适用于 B2C 出库流程，涵盖从上游订单下达到最终发运的全链路操作，包括：出库订单（上游推单 / 手动建单）→ 出库包裹 → 加入波次 → 拣货 → 复核 → 称重 → 发运。各环节需在 **PC 端** 和 **PDA 端** 协同完成。

## 二、前置条件

- **系统配置要求**：已完成作业流程、操作配置、波次规则、承运方案等策略配置。
- **设备要求**：PC 端（波次管理、打印）+ PDA 端（拣货、复核、称重）。
- **硬件要求**：PDA 设备、蓝牙秤（称重环节）、打印机（面单 / 拣货单 / 发货单）。
- **前置条件**：上游已完成出库订单推送，出库包裹状态为 **“待处理”**。

## 三、操作入口

各功能在系统中的路径如下（若系统版本不同，请以实际菜单路径为准）：

| 功能模块 | 路径 |
|----------|------|
| 出库包裹处理 | 出库 → 出库包裹 |
| 加入波次 | 出库 → 加入波次 |
| 波次管理 | 出库 → 波次管理 |
| 拣货任务 | 出库 → 拣货任务 或 PDA → 散单拣货 |
| 复核 | PC 端 → 散单复核 |
| 称重 | PC 端 → 称重模块 |
| 异常单管理 | 出库 → 异常单管理 |
| 回库明细 | 出库 → 回库明细 |

## 四、操作步骤

### 4.1 场景一：出库包裹处理

1. 系统自动预处理（无需人工干预）：
   - 根据预设规则自动 **拆合包**（查询拆包规则）。
   - 确定 **作业流程**（后续执行的复核、打包、称重环节）。
   - 分配 **承运公司**（根据承运方案）。
   - 自动 **取单**（查询承运商配置取号）。
   - 自动获取 **推荐包装方案**（上游指定 / 指定货品结构 / 推荐惯用 / 最新包装方案等多种逻辑）。
   - 定义 **时效产品**（查询时效产品管理）。

2. 手动可选操作（如有需要可执行）：
   - 手动拆合包、变更承运公司 / 产品类型 / 增值服务、变更作业流程。
   - 重新获取面单、获取推荐包装方案、指定包装方案。
   - **补货分析**：对比包裹需求数量与拣选区可用数量，缺货时可手动创建从存储区到拣选区的补货任务。
   - 如不需要取号或已有可用单号，可编辑录入单号（自提单手动录入自提单号用于交接扫描）。

   ![](./images/01_创建仓库、货主、员工_操作说明书_10_e94b96dc.png)

::: tip 提示
完成取号且库存充足后即可加入波次。
:::

::: warning 注意事项
**作废**：卡单后可作废。配置节点允许在复核后作废，直接返架。
:::

### 4.2 场景二：加入波次

1. **筛选订单加入波次**：在出库包裹页面筛选待处理的包裹，点击 **“加入波次”**。

   ![](./images/01_创建仓库、货主、员工_操作说明书_11_e213f0ae.png)

2. **包裹汇单，选择波次规则，组波**：选择波次规则，可使用预分配功能根据库区区域汇波。

   ![](./images/01_创建仓库、货主、员工_操作说明书_12_4b5dcb0d.png)
   ![](./images/01_创建仓库、货主、员工_操作说明书_13_2ccb6f1a.png)

3. **波次汇单 — 所见即所得**：波次汇单页面可看到创建波次结果，波次会自动运行：分配 → 分组 → 创建拣货任务。

   ![](./images/01_创建仓库、货主、员工_操作说明书_14_457f7d24.png)

::: tip 提示

### 4.3 场景三：拣货任务执行

#### (1) 方式一：纸质单拣货

1. 打印拣货单。
2. 线下完成拣货后，回到电脑端确认拣货任务。

   ![](./images/01_创建仓库、货主、员工_操作说明书_16_ef5e73cc.png)

#### (2) 方式二：PDA拣货（推荐）

1. 在 PDA 上 **“散单拣货”** → **领取拣货任务**（可分配给某人，也可自由领取）。
2. 扫描 **库位**、**货品编码**（不区分大小写）。
3. 输入数量（如货品没有贴条码则手动输入货品条码）。
4. 点击 **提交**。
5. 继续未完成任务明细，直到全部完成。

## 五、操作结果

- **出库包裹**：完成所有流程节点后，状态变为 **“已发运”**，并将状态回传上游。
- **波次**：释放后生成拣货任务，拣货完成后状态更新。
- **复核**：复核通过后进入打包环节。
- **称重**：称重成功后自动发运（根据流程配置）。

## 六、注意事项

### 6.1 重点提醒

::: danger 重点提醒
- **拦截**：订单拦截后会进入异常单管理列表，异常确认后会取消拦截单。不同节点拦截后的处理方式不同，具体见下方表格。
- **回库**：如包裹已下架（已完成拣货），作废后会进入回库明细列表。需在回库明细中确认回库，将货品返回库位。
- **取号失败**：常见原因包括余额不足、地址不详细、承运商配置错误等，请对照常见问题中的异常表格处理。
- **称重校验**：录入实际称重与理论重量差异太大时，需在仓库货主配置中修改重量校验参数。
- **导出限制**：出库包裹导出每次限制 **2000** 条，如需更多数据请移步 **报表→出库包裹明细**。
:::

### 6.2 一般提醒

::: warning 注意事项
- 波次生成后默认锁定，需手动释放才能执行拣选任务。
- PDA 操作规则可在【配置-操作配置-拣货】设置是否校验库位、货品编码、数量，以及是否首件扫描、逐件扫描。

### 7.2 高频 FAQ

**Q1：批量波次和散单波次有什么区别？**

A：批量波次适用于货品结构一致的订单，可批量拣货、批量复核、批量称重，效率更高。散单波次适用于无规律的订单，需要逐单复核称重。

**Q2：为什么波次已生成但 PDA 看不到拣货任务？**

A：波次生成后默认处于锁定状态，需要在波次管理页点击 **“释放波次”**，释放后 PDA 端才能看到拣货任务。

**Q3：复核有哪几种模式，如何选择？**

A：普通复核（散单波次）、批量复核（批量波次）、货找单复核（单品单件波次，配合后置打单）、播种复核（总拣后需二次分拣的场景）。

**Q4：多波次总拣是什么？**

A：将多个波次合并到一起拣选，拣选批量更大，更高效，适用于贴单即发场景。

**Q5：前置打单和后置打单有什么区别？**

A：前置打单适用于需要面单分拣的作业（如边拣边分），后置打单适用于理论包裹数不明确、复核时才能确定包裹数的场景（如单品单件混品波次、货找单复核）。
