# 创建库位、SKU

## 一、适用场景

仓库完成基础组织架构配置后，需要创建物理库位来存放货物，并管理货品基础档案（SKU）。库位是货物的存放容器，入库上架和出库拣货都需精确到库位。SKU是货物的数字档案，包含条码、规格、效期等关键信息。**库位规划**直接影响入库定位和出库分配效率，**SKU档案的完整性**决定了收货、质检、效期管理等作业能否正常运行。

## 二、前置条件

- **账号权限**：拥有仓库管理员角色，可访问“系统管理”模块。
- **数据准备**：已完成仓库、货主、员工的创建；明确库位与货架物理布局；准备好库位编码方案。
- **材料准备**：如需迁仓，请准备 **《库位配置导入.xlsx》** 模板；如需更新现有库位，请准备 **《库位配置更新模版.xlsx》**。模板可在钉钉文档的附件中下载。

## 三、操作入口

- **创建库位**：**系统管理** → **基础资料** → **库位管理**
- **创建SKU**：**系统管理** → **基础资料** → **SKU档案**

## 四、操作步骤

### 4.1 场景一：创建库位

1. **添加货架**
   在库位管理界面，找到货架管理功能，按实际物理布局录入货架信息（编码、名称、区域等）。
   ![添加货架](./images/01_创建仓库、货主、员工_操作说明书_10_e94b96dc.png)

2. **添加库区**
   在货架下创建库区，如拣选区、存储区。
   ![添加库区](./images/01_创建仓库、货主、员工_操作说明书_11_e213f0ae.png)

3. **创建库位**
   有两种方式可选：
   - **方式一：导入库位**（推荐迁仓场景，保留原有库位编码）
     下载 **《库位配置导入.xlsx》** 模板，按格式填写后导入。查看导入结果：
     ![导入库位](./images/01_创建仓库、货主、员工_操作说明书_12_4b5dcb0d.png)
     ![导入结果](./images/01_创建仓库、货主、员工_操作说明书_13_2ccb6f1a.png)
   - **方式二：手动添加库位**（推荐开新仓场景，系统自动编码）
     ![手动添加库位](./images/01_创建仓库、货主、员工_操作说明书_14_457f7d24.png)

   ::: warning 库位参数说明
   创建库位时需注意以下关键参数，按业务需求配置：

   | 参数 | 说明 |
   |------|------|
   | **库位类型** | 统一选择“库存”。目前分配逻辑只分配库存类型库位；收货、发货、加工类型需移库到库存类型。 |
   | **库存种类** | 用于限制库位的业务类型，可作为库位冻结使用。 |
   | **忽略容量** | 开启后，入库定位时不会因库位剩余容量不足而跳过该库位。 |
   | **巷道编码** | 填写后可应用于：包裹汇单筛选、波次规则按巷道创建拣选任务。 |
   | **存储策略** | 关联后可限制库位可存入货品的属性（效期、品类数、ABC分类、批次号等）。 |
   | **动线号** | 上架、拣货、盘点动线号：用于在任务明细中排序库位动线。 |
   | **绑定货主/绑定SKU** | 绑定后，定位/分配环节会优先分配该库位（若容量不足则就近找空库位）。 |

   *详细配置界面可参考系统内截图。*
   :::

### 4.2 场景二：创建SKU

1. **方式一：上游通过接口同步SKU（主要方式）**
   上游OMS系统通过接口推送SKU档案到WMS。这是标准做法，保证数据一致性。无需在WMS手动操作。

2. **方式二：手动创建SKU（应急方式）**
   当无法及时同步时，可手动创建SKU应急收货。但**不建议长期使用**，应尽快由OMS推送。

3. **首次入库新品维护**
   若仓库货主配置中开启了**收货提醒**，则首次入库时会触发新品维护流程。
   ![新品维护1](./images/01_创建仓库、货主、员工_操作说明书_15_7567dfc0.png)
   ![新品维护2](./images/01_创建仓库、货主、员工_操作说明书_16_ef5e73cc.png)
   ![新品维护3](./images/01_创建仓库、货主、员工_操作说明书_17_9cc56e8f.png)

   ::: warning SKU关键配置参数
   手动创建或维护SKU时，请注意以下字段：

   | 参数 | 说明 |
   |------|------|
   | **行业属性** | 设置收货时需要采集的信息。 |
   | **类别、ABC分类** | 用于与库位属性匹配，辅助定位分配。 |
   | **单位尺寸** | 用于推荐包材、校验库位可存储数量。 |
   | **托规** | 根据托规自动拆分收货明细并创建多条上架任务明细，每次上架生成一个虚拟托盘码。 |
   | **包装单位条码** | 扫描后可关联SKU编码，但不会记录到收货明细中，仅作为便捷操作。 |
   | **序列号管理/序列号规则** | 是否管理序列号、在哪些环节管理、序列号截取规则。其他配置在“仓库货主配置”中设置。 |
   | **效期相关字段** | 失效日期 - 临期预警天数 = 临期日期；失效日期 - 禁售天数 = 禁售日期。库存到达临期日期自动变为临期状态，到达禁售日期自动变为禁售状态。 |
   :::

   ::: tip 条码截取规则
   在SKU档案中可引用**条码截取规则**，支持灵活截取条码，适用于冷链快递前置仓等场景。PC/PDA作业扫描条码时，按规则截取后匹配条码档案，即扫描成功。该功能已支持：收货、上架、散单拣货、批量拣货、多波次拣货、库存查询、直接移库等作业。
   ![条码截取规则截图](./images/01_创建仓库、货主、员工_操作说明书_18_42ee3433.png)
   ![SKU引用示例](./images/01_创建仓库、货主、员工_操作说明书_19_730c34e3.png)
   :::

## 五、操作结果

- **库位创建成功**：库位列表中出现新库位，状态正常。后续入库定位、出库分配均可使用。
- **SKU创建成功**：SKU档案中新增记录，可用于收发货、库存查询、效期管理等操作。若通过接口同步，WMS与OMS数据一致。

## 六、注意事项

- **库位类型**必须选择“库存”，否则分配逻辑无法识别。
- 手动创建SKU仅作应急，**强烈建议**由上游OMS统一推送，避免数据不一致。
- 开启**忽略容量**后，入库定位不会因容量满而跳过该库位，请按实际管理需要审慎设置。
- **存储策略**配置后，入库时会校验货品属性与库位要求是否匹配，不匹配则无法上架。
- 首次入库新品维护需在**仓库货主配置**中开启“收货提醒”，否则不会触发。
- 效期相关字段（临期预警、禁售、禁收）需在SKU中正确填写，系统会根据公式自动计算状态和拦截逻辑。

## 七、常见问题

### 7.1 Q1：库位类型如何选择？
统一选择“库存”类型。目前分配逻辑只分配库存类型库位。收货、发货、加工类型库位需要移库到库存类型才能正常使用。

### 7.2 Q2：什么时候使用导入库位，什么时候手动添加？
- **迁仓场景**（保留原有库位编码）使用**导入**功能。
- **开新仓场景**使用**手动添加**，系统自动编码。

### 7.3 Q3：SKU应该由WMS创建还是由OMS推送？
**推荐由上游OMS通过接口同步**，保证数据一致性。WMS手动创建SKU仅作为应急手段，能保证先把货收进来，但尽量不用。

### 7.4 Q4：效期预警、禁售、禁收天数如何使用？
- 失效日期 - 临期预警天数 = 临期日期
- 失效日期 - 禁售天数 = 禁售日期
- 库存到达临期日期自动变为**临期状态**，需前往调整单据中确认调整。
- 库存到达禁售日期自动变为**禁售状态**，需前往调整单据中确认调整。
- 入库日期距失效日期小于**禁收天数**则不能收货。
- 入库日期距失效日期小于**收货预警天数**，收货时会提醒但不阻断。

（相关界面截图请参见步骤中的效期参数图）

### 7.5 Q5：条码截取规则如何使用？
作业扫描条码支持灵活截取。在SKU档案中引用条码截取规则后，PC/PDA扫描时按规则截取，若截取后的内容等于条码档案中的某一条码，则扫描成功。现已支持收货、上架、拣货、移库等主要作业。

### 7.6 常见异常及处理

| 异常现象 | 常见原因 | 解决方案 |
|----------|----------|----------|
| 导入库位失败/部分失败 | 导入模板格式不正确或数据缺失 | 检查模板是否按格式填写，必填项是否完整 |
| 入库定位不到库位 | 库位类型未设为“库存”或库位容量已满 | 确认库位类型为“库存”；如容量限制可开启“忽略容量” |
| 收货提醒触发但无法维护新品 | SKU未创建或仓库货主配置未开启收货提醒 | 先手动创建SKU应急，同时检查收货提醒开关 |
