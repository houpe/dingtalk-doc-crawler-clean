# 一. 创建库位、SKU

<strong>📌 文档基本信息</strong>

本文档适用于仓库管理员及操作人员，介绍在WMS系统中创建库位和SKU档案的标准操作流程。

## 1.1 业务场景与名词解释

### 1.1.1 业务场景（为什么用？）

1. 仓库完成基础组织架构配置后，需要创建物理库位来存放货物并管理货品基础档案（SKU）。库位是货物的存放容器，入库上架和出库拣货都需要精确到库位。SKU是货物的数字档案，包含条码、规格、效期等关键信息。
2. 库位的正确规划直接影响入库定位和出库分配的效率；SKU档案的完整性决定了收货、质检、效期管理等作业环节能否正常运行。

### 1.1.2 核心名词解释（不迷路）

- <strong>库位</strong>：货物的物理存放位置，是仓库中最小的存储单元。
- <strong>库区</strong>：多个库位的逻辑分组，如拣选区、存储区。
- <strong>货架</strong>：库区的物理载体，库区归属于货架。
- <strong>SKU</strong>：Stock Keeping Unit，商品档案/库存保有单位，是货物的数字档案。
- <strong>库位类型</strong>：统一选择"库存"类型，目前分配逻辑中只分配库存类型库位。收货/发货/加工类型需要移库到库存类型。
- <strong>库存种类</strong>：对库位的业务类型限制，可作为库位冻结使用。
- <strong>忽略容量</strong>：开启后入库定位时不会因库位剩余容量不足而跳过该库位。
- <strong>存储策略</strong>：限制库位可存入货品属性，包括效期、品类数、ABC分类、批次号等。

## 1.2 前置准备与环境配置

- <strong>前置条件</strong>：已完成仓库、货主、员工的创建。明确库位与货架物理布局。准备好库位编码方案。
- <strong>准备材料</strong>：如需迁仓，请准备《库位配置导入.xlsx》模板。如需更新现有库位，请准备《库位配置更新模版.xlsx》。

#### 1.3.1.4 库位相关参数说明

| <strong>库位参数</strong> | <strong>意义</strong> |
|----------------|----------|
| 库位类型 | 统一选择"库存"，目前分配逻辑中只分配库存类型库位。收货、发货、加工类型需要移库到库存类型。 |
| 库存种类 | 用于对库位整库位限制业务类型。可以作为库位冻结使用。可以对特定库位库存特殊限制。 |
| 忽略容量 | 忽略容量后，就不会在入库定位时因库位剩余容量不足而跳过该库位。 |
| 巷道编码 | 填了巷道编码可以应用在：包裹汇单筛选包裹、波次规则按巷道分别创建拣选任务。 ![](./images/01_创建仓库、货主、员工_操作说明书_15_7567dfc0.png)![](./images/01_创建仓库、货主、员工_操作说明书_16_ef5e73cc.png) |
| 存储策略 | 关联存储策略后，可限制库位可存入货品属性。货品的多种属性包括效期、品类数、ABC分配、批次号等。范围控制：存入库位的库存范围。 ![](./images/01_创建仓库、货主、员工_操作说明书_17_9cc56e8f.png) |
| 动线号 | 上架动线号、拣货动线号、盘点动线号：用于在任务明细中排序库位动线。 |
| 绑定货主/绑定SKU | 绑定货主、SKU，会在定位、分配环节优先分配。如收货定位优选绑定库位，如绑定库位剩余容量不足，则会就近寻找空库位。 ![](./images/01_创建仓库、货主、员工_操作说明书_18_42ee3433.png) |

## 1.4 常见异常与兜底方案（卡住了怎么办？）

| <strong>序号</strong> | <strong>异常现象</strong> | <strong>常见原因</strong> | <strong>解决方案</strong> |
|----------|----------------|----------------|----------------|
| 1 | 导入库位失败/部分失败 | 导入模板格式不正确或数据缺失 | 检查导入模板是否按格式填写，必填项是否填写完整 |
| 2 | 入库定位不到库位 | 库位类型未设为"库存"或库位容量已满 | 确认库位类型为"库存"；如容量限制可开启"忽略容量" |
| 3 | 收货提醒触发但无法维护新品 | SKU未创建或仓库货主配置未开启收货提醒 | 先手动创建SKU应急，同时检查收货提醒开关 |

## 1.5 高频常见问题（FAQ）

3. <strong>Q1：库位类型如何选择？</strong>

<strong>A</strong>：统一选择"库存"类型。目前分配逻辑中只分配库存类型库位。收货、发货、加工类型库位需要移库到库存类型才能正常使用。

4. <strong>Q2：什么时候使用导入库位，什么时候手动添加？</strong>

<strong>A</strong>：迁仓场景（保留原有库位编码）使用导入功能。开新仓场景使用手动添加，系统会自动编码。

5. <strong>Q3：SKU应该由WMS创建还是由OMS推送？</strong>

<strong>A</strong>：推荐由上游OMS通过接口同步SKU到WMS，保证数据一致性。WMS手动创建SKU仅作为应急手段，能保证先把货收进来。

1. <strong>Q4：SKU 临期预警天数、禁售天数、收货预警天数如何使用？</strong>

A：

失效日期-临期预警天数=临期日期

失效日期-禁售天数=禁售日期

库位库存到达临期日期自动转变为临期状态，前往调整单据中确认调整

库位库存到达禁售日期自动转变为禁售状态，前往调整单据中确认调整

![](./images/02_创建库位、SKU_操作说明书_12_df33ca37.png)

![](./images/02_创建库位、SKU_操作说明书_13_72da43aa.png)

入库日期距离失效日期小于禁收天数则不能收货

![](./images/02_创建库位、SKU_操作说明书_14_541ce010.png)

入库日期距离失效日期小于收货预警天数 ，收货时会提醒，但不阻断
