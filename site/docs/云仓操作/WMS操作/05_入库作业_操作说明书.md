# 入库作业

::: tip 文档基本信息
本文档覆盖WMS入库全流程：上游推单/本地建单 → PC/PDA收货扫描 → 收货记录拆分与自动定位 → 上架任务（纸质单/PDA） → 容器收货 → 期初收货（切仓平移）。
:::

## 一、适用场景
库房操作人员需要将外部货物收入仓库并上架到指定库位时，按本文档操作。包括以下场景：
- 上游OMS推单或本地手动建单
- PC端或PDA端收货扫描
- 上架任务执行（纸质单或PDA）
- 容器收货（补充功能）
- 期初收货（仅用于仓库切换）

## 二、前置条件
- **权限要求**：具备仓库操作员权限，拥有入库相关菜单的访问权限。
- **系统配置**：已完成**定位规则**、**定位条件**、**上架策略**等入库策略配置（参考《配置规则策略》）。已完成**库位**创建（参考《创建库位、SKU》）。
- **设备要求**：PC端用于收货扫描和上架任务管理；PDA端用于移动收货和上架操作；打印机用于打印入库单和上架任务单。
- **前置数据**：SKU档案已维护完整（条码、效期、批次属性等）；存储区库位已就绪。

## 三、操作入口
- **入库单据**：入库 → 入库单据
- **收货扫描**（PC）：入库 → 入库单据 或 收货扫描
- **普通收货**（PDA）：PDA端 → **普通收货**
- **上架任务**（PC）：入库 → 上架任务
- **上架任务**（PDA）：PDA端 → **上架任务**

## 四、操作步骤

### 4.1 场景一：新建入库单
1. **上游推单（推荐）**：上游OMS系统通过标准接口推送入库单据到WMS。入库单自动进入【入库-入库单据】列表，可直接开始收货。
2. **本地手动建单**（当客户未对接系统时）：进入**入库单据**页面，点击**新建入库单**，选择货主、填写入库明细后提交。
   ![](./images/01_创建仓库、货主、员工_操作说明书_10_e94b96dc.png)

### 4.2 场景二：PC收货扫描
1. **进入收货扫描页面**，有两种方式：
   - 方式一：点击入库单列表中对应单据的**开始收货**按钮，直接进入。
     ![](./images/01_创建仓库、货主、员工_操作说明书_11_e213f0ae.png)
   - 方式二：打印入库单，在**收货扫描**页面扫描入库单号，进入。
     ![](./images/01_创建仓库、货主、员工_操作说明书_12_4b5dcb0d.png)
2. **执行扫描**：
   - **按件扫描**：扫一次添加已扫描明细1件，适用于逐件核对的高精度收货。
   - **批量扫描**：扫描商品条码，手动输入数量，提交后一次性记录多件，适用于大批量收货。
   - **批量完成扫描**：勾选入库单明细行，点击**批量完成扫描**，所有勾选行全部加入已扫描明细，提交收货。适用于整单快速收货。
   ![](./images/01_创建仓库、货主、员工_操作说明书_13_2ccb6f1a.png)
   ![](./images/01_创建仓库、货主、员工_操作说明书_14_457f7d24.png)

### 4.3 场景三：PDA普通收货
1. **进入PDA端 → 普通收货**。
   ![](./images/01_创建仓库、货主、员工_操作说明书_15_7567dfc0.png)
2. **选择扫描模式**：
   - **按件模式**：切换到**按件**，逐件扫描，扫描一次添加1件。适用于高精度收货。
   - **批量模式**：切换到**批量**，扫描一件商品后手动输入数量，一次性记录多件。适用于大批量收货。
   - **全选批量完成**：全选明细行，批量完成扫描。
     ::: danger 重点提醒
     如果SKU需要采集**批次/效期**属性，则不允许批量完成扫描，需逐条录入属性信息后提交。
     :::
   ![](./images/01_创建仓库、货主、员工_操作说明书_16_ef5e73cc.png)
3. **快速上架（可选）**：收货后立即进入上架环节，扫描或选择推荐库位，立即完成上架。省去先收货再领取上架任务的两步操作。
   ![](./images/01_创建仓库、货主、员工_操作说明书_17_9cc56e8f.png)

### 4.4 场景四：收货记录与自动定位（系统自动处理）
1. **拆分收货记录**：系统根据库存属性（**批次、批号、效期、SKU档案一级单位托规**）自动拆分为多条收货记录。每条收货记录对应一条上架任务明细和一条库位库存明细。
2. **自动定位库位**：系统自动调用定位条件和定位规则，结合库位配置（区域绑定货主优先、空库位优先/补满库位优先），确定每条收货记录的上架目标库位。
3. **创建上架任务**：系统自动依据定位结果创建上架任务，任务出现在【入库-上架任务】列表中。
   ![](./images/01_创建仓库、货主、员工_操作说明书_18_42ee3433.png)

### 4.5 场景五：上架任务执行
**系统功能路径**：PC端：入库 → 上架任务；PDA端：上架任务

#### (1) 上架任务操作
1. **分配任务**：在PC端上架任务列表，选中任务点击**分配**，可指定PDA用户执行，其他用户不能领取该任务。
2. **取消分配**：取消后，用户可在PDA自由领取任务，或重新分配给其他人。
3. **取消任务**：取消后，关联的收货明细被重置为待定位状态。可在**收货记录**页面手动重新定位库位并创建上架任务。
4. **完成上架**，有两种方式：

**方式一：纸质单上架（适用于小型仓库）**
1. 打印纸质上架单。
   ![](./images/01_创建仓库、货主、员工_操作说明书_19_730c34e3.png)
2. 仓库人员凭纸质单线下完成上架，若有调整上架库位或数量，记录在纸质单上。
3. 回到电脑端**任务确认**：根据纸质单记录的实际作业结果，在电脑端完成系统上架。

**方式二：PDA上架（适用于大型仓库）**
- 任务可以分配给员工，员工在PDA领取**待上架**任务，按PDA指引扫描上架。
  ![](./images/01_创建仓库、货主、员工_操作说明书_1_68925fe0.png)
  ![](./images/01_创建仓库、货主、员工_操作说明书_20_e59e47f8.png)

### 4.6 场景六：容器收货（补充功能）
**说明**：收货时扫描容器，将货物收入容器中，后续通过扫描容器领取上架任务。实际使用较少，仅适用于有容器管理需求的仓库。
1. **收货到容器**：在PDA端选择**容器收货**，扫描容器条码。
   ![](./images/01_创建仓库、货主、员工_操作说明书_2_5e45b2a3.png)
2. **批量收货或逐件收货**。
   ![](./images/01_创建仓库、货主、员工_操作说明书_3_925b7af9.png)
   ![](./images/01_创建仓库、货主、员工_操作说明书_4_211d769e.png)
   ![](./images/01_创建仓库、货主、员工_操作说明书_5_1bc99e39.png)
3. **扫描容器上架**：
   - 扫描容器查询任务 → 领取任务 → 扫描SKU、录入数量 → 上架。
     ![](./images/01_创建仓库、货主、员工_操作说明书_6_464c1e9c.png)
   - 上架后容器释放，可重新用于下次收货。
     ![](./images/01_创建仓库、货主、员工_操作说明书_7_1a9ea66b.png)
     ![](./images/01_创建仓库、货主、员工_操作说明书_8_11d55e9c.png)

### 4.7 场景七：期初收货（仅切仓使用）
**说明**：仓库切换过程中从旧WMS系统平移库存的特殊收货方式。可跳过SKU收货的多种校验规则（条码校验、效期校验、批次校验等），一键收货。**仅用于切仓场景，日常操作严禁使用**。
- 在系统【入库-入库单据】页面，选择期初收货类型的入库单，点击**期初收货**完成操作。
  ![](./images/01_创建仓库、货主、员工_操作说明书_9_c1a2da40.png)

## 五、操作结果
- 入库单状态变为**已收货**或**已上架**。
- 收货记录、上架任务自动生成，库位库存明细更新。
- 对于PDA快速上架，收货即上架，状态直接为已上架。
- 期初收货后库存直接平移至新系统。

## 六、注意事项
- 手动建单仅用于客户未对接系统的应急场景，上游OMS已对接时统一使用接口推单。
- 需要采集批次/效期属性的SKU，**不允许**使用批量完成扫描，必须逐条录入属性信息。
- PDA上架任务如需指定专人执行，先在PC端**分配**任务。
- 期初收货不可用于日常入库，否则可能导致库存数据混乱。
- 上架任务取消后，关联收货记录重置为待定位状态，需手动重新定位。

## 七、常见问题

| 序号 | 异常现象 | 常见原因 | 解决方案 |
|------|----------|----------|----------|
| 1 | 收货时无法扫描货品条码 | SKU档案中未维护条码 / 操作配置校验开关限制 | 检查SKU档案中的**条码**字段；检查【配置-操作配置-收货】中的扫描校验设置 |
| 2 | 收货后没有生成上架任务 | 定位规则未配置或定位条件未关联 | 检查【配置-定位规则】和【配置-定位条件】，确保入库单据类型已关联定位规则 |
| 3 | 上架任务无法领取 | 任务已被分配给其他用户 | 在PC端**取消任务分配**，或由已分配用户执行。也可取消任务后重新定位并创建任务 |
| 4 | PDA收货无法批量完成扫描 | SKU需要采集**批次/效期**属性 | 逐条录入属性信息后提交，不允许批量完成 |
| 5 | 上架时库位不足 | 库位容量已满且未开启忽略容量 | 在库位管理中开启**忽略容量**选项，或手动将库位类型切换为**库存**类型 |

### 7.1 FAQ
**Q1：PC收货扫描和PDA普通收货有什么区别？**
A：功能基本一致，都支持按件/批量扫描。PC端适合集中收货场景；PDA端适合移动收货，支持快速上架。大仓库推荐PDA操作。

**Q2：什么时候使用上游推单，什么时候手动建单？**
A：上游OMS已对接时，统一使用接口推单，保证数据一致性和可追溯性。手动建单仅用于客户未对接系统的应急场景。

**Q3：收货记录为什么会拆分成多条？**
A：系统根据库存属性（批次、批号、效期）和SKU档案的一级单位托规自动拆分。同一SKU不同批次会拆成多条记录，对应多条上架任务和库位库存明细，保证库存维度精确。

**Q4：取消上架任务后会怎样？**
A：取消任务后，关联的收货明细被重置为**待定位**状态。可在【入库-收货记录】页面手动重新定位库位并重新创建上架任务。

**Q5：期初收货和普通收货有什么区别？**
A：期初收货可跳过SKU收货的多种校验规则（条码/效期/批次校验等），一键收货。仅用于切仓平移库存，日常入库严禁使用期初收货。
