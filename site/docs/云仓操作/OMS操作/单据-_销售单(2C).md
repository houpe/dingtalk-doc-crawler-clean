## 1.3 场景化标准操作步骤（怎么用？）

### 1.3.1 场景一：[ERP客户>出库单(2B)]

- <strong>系统功能路径</strong>：`登录系统` -> `进入左侧菜单栏` -> `[OMS订单中心]` -> `[出库管理]` -> `[销售单(2C)]`
- <strong>快捷直达链接</strong>：👉 [[销售单（2C）-鲸天系统](https://wms.ztocc.com/app/#/send/sellOrder)]

#### 1.3.1.1 核心操作步骤

1. <strong>[ERP同步销售单(2C)->OMS]</strong>ERP客户：ERP中创建销售单(2C)同步销售单(2C)到OMS![image.png](./images/单据-_入库单_2_9b4425c3.png)
2. <strong>[WMS对接]</strong> OMS接收销售单(2C)->自动同步到WMS中
3. <strong>[入库单回传]</strong> WMS销售单(2C)出库完成->自动回传出库信息到OMS中->OMS回传出库信息到ERP

### 1.3.2 场景二：[非ERP客户>出库单(2B)]

- <strong>系统功能路径</strong>：`登录系统` -> `进入左侧菜单栏` -> `[OMS订单中心]` -> `[出库管理]` -> `[销售单(2C)]`
- <strong>快捷直达链接</strong>：👉 [[销售单（2C）-鲸天系统](https://wms.ztocc.com/app/#/send/sellOrder)]

#### 1.3.2.1 核心操作步骤

1. <strong>[OMS创建出库单(2B)]</strong>1.1，客户在OMS中创建销售单(2C)/导入销售单(2C)，点击新增/导入1.2，按照要求填写信息：头表信息：货主，仓库，单据类型，外部单号，物流公司，物流单号，备注，联系人，联系电话，收件省，收件市，收件区，详细地址明细信息：商品，商家系统编码，商家商品编码，总数量，二级单位，最小单位1.3，新增/导入文件中必填信息填写完成保存，回到销售单(2C)列表![image.png](./images/单据-_入库单_3_c7162966.png)
2. <strong>[WMS对接]</strong> 2.1 【<strong>审核</strong>】OMS审核销售单(2C)->自动同步到WMS中![image.png](./images/单据-_入库单_4_38169c00.png)2.2 【编辑】新增完成后若需调整信息则点击编辑重新编辑销售单(2C)2.3 【复制】若需创建相同类型的订单时刻复制新增/导入的销售单(2C)，复制后支持重新编辑销售单(2C)信息2.4 【取消】若创建入库单有问题则可点击取消按钮，取消销售单(2C)
3. <strong>[入库单回传]</strong> WMS销售单(2C)出库完成->自动回传出库信息到OMS中->OMS回传出库信息到ERP

## 1.5 高频常见问题（FAQ）

- <strong>Q1：[导入时提示商品二级单位未维护]</strong>
- <strong>A</strong>：[需要在WMS中维护商品单位，维护后商品单位会自动同步到OMS中，可重新导入入库单]
- <strong>Q2：[导入时提示单据【xxx】已存在]</strong>
- <strong>A</strong>：[当前导入的货主单据号已存在，不能够重复导入，需要更新单据号之后重新导入]
- <strong>Q3：[导入时提示商品【xxx】不属于货主【xxx】]</strong>
- <strong>A</strong>：[导入的商品明细中的商品不是导入货主的商品，需要检查导入数据信息，更新后重新导入]
- <strong>Q4：[ERP下发单据/新增单据审核后OMS单据状态为什么是缺货，看OMS中库存足够]</strong>
- <strong>A</strong>：[OMS仓库开启库存管理后，在OMS接单/审核后会校验可用库存是否足够，若不足则更新单据状态为缺货，不推送WMS，在校验OMS库存是否足够时是校验可用库存，而不是总库存]
- <strong>Q5：[缺货状态下单据后如何处理]</strong>
- <strong>A</strong>：[与客户确认是否继续出货，若是则通知客户补货后重新审核，则可将单据同步到WMS继续出货，若不继续出货则取消单据]
