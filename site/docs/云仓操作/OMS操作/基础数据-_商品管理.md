# 基础数据商品管理

::: tip 文档基本信息
本文档说明商品管理的操作流程，包括非ERP客户手动新增和ERP客户系统同步两种场景。
:::

## 一、适用场景
- **ERP客户**：已完成ERP对接，由ERP创建商品基础资料并自动同步到OMS。
- **非ERP客户**：未对接ERP，需手动在OMS创建商品资料。

## 二、前置条件
- 拥有 **OMS订单中心** 的菜单权限。
- 非ERP客户：准备商品相关信息（如货主、商品名称、条形码、品牌等）。
- ERP客户：确保ERP中已维护商品基础资料。

## 三、操作入口
- **系统功能路径**：`登录系统` → `左侧菜单栏` → `[OMS订单中心]` → `[基础数据]` → `[商品管理]`
- **快捷直达链接**：👉 [商品管理页面](https://wms.ztocc.com/app/#/base/goods)

## 四、操作步骤

### 4.1 场景一：新增商品（非ERP客户）
1. 进入 **商品管理** 页面，点击 **新增** 按钮。
2. 按页面要求填写信息，带 **\*** 的字段为必填项：
   - 货主、商品名称、商家商品编码、条形码、品牌、品类、产地、商品成本、商品标签、商品类型、温层、允收期（天）、保质期（天）、允发期（天）、体积、毛重、净重、有无批次、匹配规则、是否易碎品、是否液体、原装发货、型号、规格、包装材料类型、包装单位。
3. 填写完成后点击 **保存**。

   ![新增商品截图](./images/单据-_入库单_1_b9ab1010.png)

### 4.2 场景二：新增商品（ERP客户）
- ERP同步商品信息到OMS，无需手动新增。OMS保存后自动同步到WMS。

::: warning 注意事项
- 标签、类型、温层等下拉选项需提前在系统配置中维护。
- 商品同步到WMS时，**包装单位**需在WMS中已维护，否则同步失败。
:::

## 五、操作结果
- 非ERP客户：保存成功后，商品列表新增一条记录，状态为 **启用**。
- ERP客户：ERP推送成功后，OMS商品列表自动出现对应商品，并自动同步至WMS。

## 六、常见问题

**Q1：商品未推送到WMS，报“商品单位不存在”**
- **A**：原因是OMS中填写的 **包装单位** 在WMS中未维护。请先在WMS的基础数据中维护对应包装单位，再重新推送。
