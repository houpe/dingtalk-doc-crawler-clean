## 1. 背景

本轮工作围绕钉钉文档整理页展开，目标是把原始文档中的：

- 标题
- 菜单
- 操作路径
- 流程图

整理成一份适合预览和后续继续清洗的网页版本，并且让提取结果尽量稳定、可读。

当前用户已经确认：现在的网页预览效果基本符合预期。

## 2. 用户已确认的需求

### 2.1 标题层级

最终整理页按下面的层级展示：

```text
分组
  标题
    菜单
      操作路径
    流程图
```

对应到 Markdown 层级是：

- `## 分组`
- `### 标题`
- `#### 菜单：...`
- `##### 操作路径：...`
- `#### 流程图`

### 2.2 菜单与操作路径

用户明确要求：

- 菜单作为二级信息展示
- 菜单下再拆出操作路径作为三级信息
- 不再用一整行 `A > B > C` 直接平铺

现在的拆分规则是：

- 第一段作为 `菜单`
- 后续段作为 `操作路径`

示例：

```text
经营管理中心 > 电子面单 > 电子面单申购
```

会展示为：

```text
菜单：经营管理中心
操作路径：电子面单 > 电子面单申购
```

### 2.3 流程图中的菜单标记

用户先提出希望在流程图步骤里标出对应菜单，随后明确选择了“方案 2”：

- 不把菜单写进每个步骤节点
- 而是在流程图中用“菜单块”包住对应步骤
- 如果相邻步骤属于同一个菜单，则合并成一个菜单块

当前 Mermaid 的实现效果类似：

```text
菜单：微信小程序
  打开下单小程序
  填写下单信息

菜单：订单查询
  查询订单轨迹
```

### 2.4 提取精度

用户明确指出早期版本存在以下问题：

- 图片处理有问题
- 内容没有按要求优化
- tree 里的菜单混乱
- 流程图和菜单提取不够精确

本轮已经重点处理的是后两项：菜单和流程图提取精度。

## 6. 当前提取策略

### 6.1 流程图提取策略

已经做了几轮收紧，当前原则是：

- 只保留明显像“操作步骤”的内容
- 说明性文档、问答型文档、规则说明文档，不强行画流程图
- 如果步骤不稳定，则输出：

```text
流程图：未提取到稳定步骤
```

已经专门规避的误提取来源：

- 目录项
- 问句标题
- “重点来了”“重要的事情说三遍”这类装饰性标题
- 公式/费率说明
- 纯说明、纯总结、纯优先级类文档

### 6.2 菜单提取策略

当前优先提取这些来源：

- `菜单路径：...`
- `路径：...`
- `微信小程序搜索【...】`
- `菜单搜索“...”`
- 一些已知稳定模式，如：
  - `基础配置-用户中心-员工管理`
  - 小程序入口
  - 订单查询入口

已经做了过滤，尽量避免把下面这些误识别成菜单：

- 带公式的说明文字
- 带“或者/接下来/请随我来”等混合说明句
- 口语句、问句、噪音行

### 6.3 步骤归属到菜单

当前做法是：

- 先找出步骤所在的文档区段
- 再用该区段内最近、最匹配的菜单作为这个步骤的菜单归属
- 如果连续几个步骤同属一个菜单，在 Mermaid 里合并成一个菜单块

这部分已经对下面这种文档做了定向优化：

- `如何引导客户在客户端下单？`

当前能得到类似结果：

- 菜单：`微信小程序`
  - 打开下单小程序
  - 填写下单信息
- 菜单：`订单查询`
  - 查询订单轨迹

## 9. 当前仍值得继续优化的地方

这部分不是阻塞项，但如果后续别的 AI 要继续做，建议优先看这里。

### 9.1 菜单名还有口语噪音

例如某些文档里仍可能出现类似：

- `找到平台 交易（订单）`
- `若商家有在使用的打单软件`

这类更像说明句，不像正式系统菜单名。用户已经认可当前效果，但后续可以继续做一轮菜单名清洗。

### 9.2 步骤归菜单仍是启发式

当前已经能覆盖一批典型文档，但不是严格结构化映射。若后续要继续提高准确率，可以考虑：

- 结合区段标题做更强的菜单绑定
- 给不同文档类型增加专门规则
- 对“菜单路径出现一次、后续多步复用”的场景增加更稳定的继承规则

### 9.3 没有菜单依据的流程仍会显示无菜单块

这是当前有意保守的行为：

- 宁可不标
- 不乱标

如果后续需要更激进的展示，需要先确认用户是否接受“推断型菜单归属”。

## 10. 如果后续 AI 继续接手，建议优先顺序

### 优先级 1

清洗菜单名，把明显不是系统菜单的说明句过滤掉或改写掉。

### 优先级 2

继续提高“步骤 -> 菜单”的映射准确率，重点盯：

- 网点操作
- 中心操作
- 云仓操作

### 优先级 3

如用户继续要求，可再做“按菜单聚合”的第二视图，即：

```text
分组 -> 菜单 -> 操作路径 -> 标题
```

但当前用户确认的主视图仍是：

```text
分组 -> 标题 -> 菜单 -> 操作路径 -> 流程图
```
